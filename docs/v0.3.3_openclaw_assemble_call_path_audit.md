# Root Cause Analysis: Why `assemble()` Stopped Being Called

**Date:** 2026-04-05
**Source:** OpenClaw source code audit (`openclaw-source/`)
**Related:** `v0.3.3_automatic_segmentation_stalled_incident.md`

---

## 1. Critical Finding: Slot Configuration Requirement

OpenClaw does NOT automatically call `assemble()` on all registered context engines. It calls `assemble()` on **exactly one** context engine -- the one resolved from `config.plugins.slots.contextEngine`.

### Resolution Logic (`src/context-engine/registry.ts` lines 411-426)

```typescript
export async function resolveContextEngine(config: OpenClawConfig): Promise<ContextEngine> {
  const slotId = config.plugins?.slots?.contextEngine ?? defaultSlotIdForKey("contextEngine");
  // defaultSlotIdForKey("contextEngine") returns "legacy"
  const entry = CONTEXT_ENGINE_REGISTRY_STATE.get(slotId);
  if (!entry) throw new Error(`Context engine "${slotId}" not registered`);
  const engine = await entry.factory();
  return wrapContextEngineWithSessionKeyCompat(engine);
}
```

**This means:**
- If `config.plugins.slots.contextEngine` is NOT set → resolves to `"legacy"` engine
- If set to `"episodic-claw"` → resolves to episodic-claw's engine
- **Only the resolved engine's `assemble()` is called**

### The Call Site (`src/agents/pi-embedded-runner/run/attempt.ts` line 1200)

```typescript
if (params.contextEngine) {
  const assembled = await assembleAttemptContextEngine({
    contextEngine: params.contextEngine,
    // ...
  });
}
```

There is exactly **one guard**: `if (params.contextEngine)`. If truthy, assemble is always called.

## 2. Why `assemble()` Stopped on April 1

### The Most Likely Cause: Config Resolution

For episodic-claw's `assemble()` to be called, the user's `openclaw.json` must contain:

```json
{
  "plugins": {
    "slots": {
      "contextEngine": "episodic-claw"
    }
  }
}
```

If this was removed, reset, or overridden, OpenClaw falls back to the `"legacy"` engine and episodic-claw's `assemble()` is never called.

### The March 30 Commit That May Have Caused This

**Commit `235908c30e`** (Mar 30): "fix: support multi-kind plugins for dual slot ownership"

This commit changed how plugins with multiple kinds (e.g., `["memory", "context-engine"]`) are handled during slot selection and disablement. Specifically:

- Before this commit: Plugin disablement logic may have been less strict
- After this commit: If episodic-claw lost the `contextEngine` slot to another plugin (or to the `"legacy"` default), it would be correctly disabled

**If episodic-claw was deployed/updated around March 30-April 1 and this commit was included, it could have caused the slot resolution to change.**

### Other Possible Causes

| Cause | Likelihood | How to Verify |
|-------|-----------|---------------|
| `plugins.slots.contextEngine` config was reset to default (`"legacy"`) | HIGH | Check `openclaw.json` on production server |
| episodic-claw was disabled via `plugins.entries.episodic-claw.enabled: false` | MEDIUM | Check config |
| Plugin registration happens AFTER `resolveContextEngine()` runs | MEDIUM | Check plugin loading order in OpenClaw startup logs |
| Another context engine plugin took the slot | LOW | Check what other context-engine plugins are installed |

## 3. What OpenClaw Expects from `assemble()`

### Interface (`src/context-engine/types.ts` lines 178-188)

```typescript
assemble(params: {
  sessionId: string;
  sessionKey?: string;
  messages: AgentMessage[];
  tokenBudget?: number;
  model?: string;
  prompt?: string;
}): Promise<AssembleResult>;
```

### Expected Return Type (`AssembleResult`)

```typescript
export interface AssembleResult {
  messages: AgentMessage[];
  prependSystemContext?: string;
  systemPromptAddition?: string;
  estimatedTokens?: number;
}
```

### episodic-claw's Current Return (from `src/index.ts`)

```typescript
return { messages: msgs, prependSystemContext, estimatedTokens };
```

This matches the expected interface. The return type is correct.

## 4. `ownsCompaction` Does NOT Affect `assemble()`

Confirmed: `ownsCompaction` is used ONLY for:
- Disabling Pi's internal auto-compaction (`pi-settings.ts` line 105)
- Guarding compaction hook behavior during overflow/timeout (`run.ts` lines 398, 417, 709)
- Determining compaction ownership in `compact.ts` line 1040

**It has zero impact on whether `assemble()` is called.**

---

## 5. `before_prompt_build` Hook Is Separate

The `before_prompt_build` hook runs AFTER `assemble()`:

1. Line 1200-1231: Context engine `assemble()` called
2. Line 1469-1523: `before_prompt_build` hooks run

These are complementary mechanisms. episodic-claw uses BOTH:
- `assemble()` for message assembly + `prependSystemContext` injection
- `before_compaction` / `after_compaction` hooks for compaction lifecycle

## 6. Recommended Fix

### Option A: Verify and Fix Config (Immediate)

Check the production `openclaw.json` for:

```json
{
  "plugins": {
    "slots": {
      "contextEngine": "episodic-claw"
    }
  }
}
```

If missing, add it and restart OpenClaw.

### Option B: Add Startup Diagnostic to episodic-claw

Add logging to confirm whether the context engine is being resolved:

```typescript
// In register(), after registerContextEngine:
console.log("[Episodic Memory] Context engine registered as 'episodic-claw'");
console.log("[Episodic Memory] Ensure config.plugins.slots.contextEngine = 'episodic-claw'");
```

### Option C: Fallback Segmentation (If Config Is Correct But Still Broken)

If the config is correct but `assemble()` is still not called, the issue may be in plugin loading order. Add a watcher-based fallback:

```typescript
// Watch for session file changes and trigger processTurn independently
api.on("session_start", async (event, ctx) => {
  // Start a polling loop that checks session file for new messages
  // and calls processTurn() when changes detected
});
```

## 7. Summary

| Question | Answer |
|----------|--------|
| Does OpenClaw call `assemble()` on all registered engines? | **No** -- only on the one resolved from `config.plugins.slots.contextEngine` |
| What is the default context engine? | `"legacy"` |
| Does `ownsCompaction` affect `assemble()`? | **No** |
| Does `before_prompt_build` replace `assemble()`? | **No** -- they run in sequence |
| Most likely cause of the April 1 outage? | Config `plugins.slots.contextEngine` not set to `"episodic-claw"` |
| Second most likely cause? | March 30 commit `235908c30e` changed plugin slot/disablement logic |
