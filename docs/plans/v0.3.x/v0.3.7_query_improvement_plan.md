# v0.3.7 修正プラン: Instant Deterministic Query Rewriting (Polyglot Heuristic)

**Date:** 2026-04-08
**Severity:** HIGH (Recall精度と速度に直結)
**Target Version:** v0.3.7

## 1. 問題の概要

### 問題: 現在のクエリは「生メッセージの結合」のため検索が全く当たらない

現在の実装（`src/retriever.ts` L143-152）では、直近5メッセージをそのまま結合して検索クエリとしています。
v0.3.7では、この「直近何メッセージを見るか」を `openclaw.plugin.json` から調整可能にし、**デフォルトを5→4** に変更します。

**入力例:**
```
assistant: <final>
[[reply_to_current]] 今日も一日お疲れさん！Qdrantの妄想もできたし...
</final>
user: System: [2026-04-08 01:31:33 GMT+7] Gateway restart config-apply ok...
```

**問題点:**
1. **ノイズ多すぎ**: `<final>`, `[[reply_to_current]]`, `System:`, タイムスタンプなどがクエリに含まれる。
2. **検索意図が不明瞭**: 会話の流れは含まれるが、「何を検索したいか」が明確でない。
3. **セマンティック意味が希薄化**: ノイズが埋め込みベクトルを汚染し、関連エピソードとの類似度が下がる。

## 2. 修正内容: Polyglot Heuristic (Arxiv準拠)

LLMによるQuery Rewritingは高精度だが遅延とAPIコストが発生するため、**Arxivのマルチリンガル検索研究（Mo et al. 2024等）で実証された「ヒューリスティックな用語抽出」**を採用します。

### 2.1. Arxivによる裏付け
*   **Mo et al. (2024) "A Survey of Conversational Search"**: 会話型検索における **heuristic term extraction** が、ノイズ除去とRecall向上に最も有効と結論づけています。
*   **CJK言語への適用**: 日本語・中国語・韓国語は分かち書きがないため、**Script-aware Regex（文字種別正規表現）** による抽出が、MeCab等の外部ツールに依存せず低遅延でノイズを除去する唯一の現実解です。

### 2.2. アルゴリズム（3段階）

**Phase 1: Aggressive Noise Removal**
*   XML/マークアップ除去: `<final>`, `</final>`, `<[^>]+>`
*   応答マーカー除去: `\[\[reply_to_current\]\]`
*   システムプレフィックス除去: `^System:\s*\[.*?\]\s*`
*   絵文字除去: `\p{Extended_Pictographic}`

**Phase 2: Polyglot Heuristic Term Extraction**
*   **英語・ラテン文字**: 3文字以上の単語を抽出。
*   **インドネシア語（Bahasa Indonesia）**: ラテン文字系として抽出し、インドネシア語ストップワードでノイズを除去。
*   **CJK言語（漢字・カタカナ・ハングル）**: 2文字以上の固有名詞・名詞を抽出。
*   **ストップワード除去**:
    *   `stopwords-iso` パッケージ（英語・日本語・インドネシア語・中国語・韓国語の包括的リスト）を使用。
    *   ユーザーが `openclaw.plugin.json` で定義した除外キーワードも追加。

**Phase 3: Query Assembly**
*   抽出された重要語をスペース区切りで結合。
*   重複除去 + 上位12語に制限（ベクトル汚染防止）。
*   **出力例:** `JLPT 申込 Qdrant Kasou`

### 2.3. 実装詳細

**ファイル:** `src/retriever.ts`

`retrieveRelevantContext()` 内のクエリ構築ロジックを `instantDeterministicRewrite()` に置き換えます。

```typescript
import * as stopwords from 'stopwords-iso';
import { EpisodicPluginConfig } from './types';

// モジュールスコープで一度だけ初期化（パフォーマンス最適化）
let STOPWORDS_CACHE: Set<string> | null = null;

type StopwordsDict = Record<string, string[] | undefined>;

function getStopwords(config?: EpisodicPluginConfig): Set<string> {
  if (STOPWORDS_CACHE) return STOPWORDS_CACHE;

  const stopwordsSet = new Set<string>();
  const dict = stopwords as unknown as StopwordsDict;

  // 1. stopwords-iso から英語・日本語・インドネシア語・中国語・韓国語を取得
  if (dict.en) dict.en.forEach((w: string) => stopwordsSet.add(w.toLowerCase()));
  if (dict.ja) dict.ja.forEach((w: string) => stopwordsSet.add(w));
  if (dict.id) dict.id.forEach((w: string) => stopwordsSet.add(w.toLowerCase()));
  if (dict.zh) dict.zh.forEach((w: string) => stopwordsSet.add(w));
  if (dict.ko) dict.ko.forEach((w: string) => stopwordsSet.add(w));

  // 2. ユーザー設定の除外キーワードを追加
  if (config?.queryExcludedKeywords) {
    config.queryExcludedKeywords.forEach(w => stopwordsSet.add(w));
  }

  STOPWORDS_CACHE = stopwordsSet;
  return stopwordsSet;
}

function instantDeterministicRewrite(messages: Message[], config?: EpisodicPluginConfig): string {
  // 1. Aggressive Noise Removal
  let cleaned = messages
    .map(m => {
      let text = extractText(m.content)
        .replace(/<final>[\s\S]*?<\/final>/g, '')
        .replace(/\[\[reply_to_current\]\]/g, '')
        .replace(/^System:\s*\[.*?\]\s*/gm, '')
        .replace(/<[^>]+>/g, '')
        .replace(/\p{Extended_Pictographic}/gu, '') // 絵文字除去
        .replace(/\s+/g, ' ')
        .trim();

      return text; // Role prefixes removed (noise for semantic search)
    })
    .filter(Boolean)
    .join('\n');

  const keywords = extractPolyglotKeywords(cleaned, config);
  return keywords.length > 0 ? keywords.join(' ') : cleaned;
}

function extractPolyglotKeywords(text: string, config?: EpisodicPluginConfig): string[] {
  const stopwords = getStopwords(config);
  const keywords = new Set<string>();

  // 2. Polyglot Heuristic Term Extraction
  // 英語・ラテン文字（3文字以上）
  const enMatches = text.match(/\b[A-Za-z]{3,}\b/g) || [];
  enMatches.forEach(w => {
    if (!stopwords.has(w.toLowerCase())) keywords.add(w);
  });

  // CJK言語（漢字・カタカナ・ハングル）2文字以上
  const cjkMatches = text.match(/[\p{Script=Han}\p{Script=Katakana}\p{Script=Hangul}]{2,}/gu) || [];
  cjkMatches.forEach(w => {
    if (!stopwords.has(w)) {
      keywords.add(w);
    }
  });

  // 3. Query Assembly (Top 12)
  return Array.from(keywords).slice(0, 12);
}
```

**ファイル:** `openclaw.plugin.json`

ユーザーが除外したいキーワードを設定できるようにします。

```json
"queryExcludedKeywords": {
  "type": "array",
  "items": { "type": "string" },
  "description": "Keywords to exclude from recall queries. Add words here to prevent them from being used in memory search (e.g., common noise words or sensitive topics)."
},
"recallQueryRecentMessageCount": {
  "type": "integer",
  "minimum": 1,
  "maximum": 12,
  "default": 4,
  "description": "How many recent messages are used to build the deterministic recall query. Default: 4."
}
```

### 🔍 Code Verification — Detail Findings

#### 実装品質評価

**Regex パターン (retriever.ts L48-53):**
| パターン | 判定 | 備考 |
|---------|------|------|
| `/<final>[\s\S]*?<\/final>/g` | ✅ | Non-greedy。ネストされた内容を正しくキャプチャ |
| `/\[\[reply_to_current\]\]/g` | ✅ | 既知マーカーの完全一致 |
| `/^System:\s*\[.*?\]\s*/gm` | ⚠️ 許容範囲 | 行頭+括弧付きタイムスタンプのみマッチ。`System:` 単体や行中出現は非マッチだが、現実の入力パターンでは問題なし |
| `/<[^>]+>/g` | ✅ | 汎用XMLタグ除去 |
| `/\p{Extended_Pictographic}/gu` | ✅ | BMP+サロゲートペア+ZWJシーケンス全カバー |
| `/\b[A-Za-z]{3,}\b/g` | ✅ | 英語/ラテン文字の3文字以上。`is`/`the` はストップワードで除去 |
| `/[\p{Script=Han}\p{Script=Katakana}\p{Script=Hangul}]{2,}/gu` | ✅ | ひらがなのみのノイズ(`ている`等)を自然に除外。漢字+カタカナ+ハングルの2文字以上 |

**ストップワードキャッシュ (retriever.ts L16-40):**
- `STOPWORDS_CACHE` はモジュールスコープの `let` 変数で、初回呼び出し時に一度だけ構築される
- `config?.queryExcludedKeywords` はキャッシュに追加されるが、**configが変わってもキャッシュは再構築されない** (プロセス内で config は不変なので実害なし)
- 5言語 (en/ja/id/zh/ko) のストップワードを union した Set

**クエリ構築フロー (retriever.ts L212-217):**
```
recentMessages = currentMessages.filter(excluded).slice(-recentMessageCount)
                 ↓
instantDeterministicRewrite(recentMessages, config)
                 ↓
Phase 1: ノイズ除去 → Phase 2: Polyglot抽出 → Phase 3: Top 12 assembly
```

- `EXCLUDED_ROLES` でシステムメッセージを事前フィルタ済み ✅
- 空クエリ時は `empty_query` reason で早期リターン ✅
