# v0.3.8 修正プラン: Recallクエリへのシステムメッセージ・思考内容混入防止

**Date:** 2026-04-09
**Severity:** HIGH (Recall精度に直接影響)
**Target Version:** v0.3.8

## 1. 問題の概要

### 現象
v0.3.7のPolyglot Heuristicクエリ書き換えは正常に動作しているが、**システムメッセージとLLMの思考内容が検索クエリに混入している。**

### 観測データ（2026-04-09 実ログ）

| 時刻 (UTC+7) | 抽出されたクエリ |
|---|---|
| 00:43:09 | `"media attached openclaw inbound jpg image jpeg send prefer message tool path"` |
| 01:19:23 | `"Gemini CLI Codex CRITICAL INSTRUCTION prioritize specific tool task generic bash commands"` |

### 真の根本原因: 2層構造

**Layer 1:** `system` ロールのメッセージがフィルタされていない
- `EXCLUDED_ROLES` に `system` が含まれていないため、システムプロンプト全文がそのままクエリ材料になる

**Layer 2:** `assistant` ロール内の `thinking` / `reasoning` ブロック（プラン初期版が見落とした部分）
- OpenClawはLLMの内部思考を `assistant` ロールの `content` 配列内に `{ type: "thinking", text: "..." }` として埋め込む（`transcript-repair.ts` から確認済み）
- `extractPlainText()` (`large-payload.ts` L17-38) は全ブロックの `.text` を無差別に抽出するため、**`assistant` ロールの中に混ざったthinkingテキストはロールフィルタを完全にすり抜ける**
- 実際のスクリーンショットの `.s4-thought` ブロックがまさにこのケース

**ロールフィルタだけでは不十分。** `extractPlainText` の修正もセットで実施しないと、`assistant` ロール経由でthinkingテキストが引き続きクエリに混入し続ける。

### 方針: OpenClaw 本体の実績ある thinking 除去ロジックをコピーする

**ファイル 1: `openclaw-source/src/shared/text/code-regions.ts`** (32行)
```typescript
export interface CodeRegion {
  start: number;
  end: number;
}

export function findCodeRegions(text: string): CodeRegion[] {
  const regions: CodeRegion[] = [];
  const fencedRe = /(^|\n)(```|~~~)[^\n]*\n[\s\S]*?(?:\n\2|$)/g;
  for (const match of text.matchAll(fencedRe)) {
    const start = (match.index ?? 0) + match[1].length;
    regions.push({ start, end: start + match[0].length - match[1].length });
  }
  const inlineRe = /`+[^`]+`+/g;
  for (const match of text.matchAll(inlineRe)) {
    const start = match.index ?? 0;
    const end = start + match[0].length;
    const insideFenced = regions.some((r) => start >= r.start && end <= r.end);
    if (!insideFenced) {
      regions.push({ start, end });
    }
  }
  regions.sort((a, b) => a.start - b.start);
  return regions;
}

export function isInsideCode(pos: number, regions: CodeRegion[]): boolean {
  return regions.some((r) => pos >= r.start && pos < r.end);
}
```

export type ReasoningTagTrim = "none" | "start" | "both";

const QUICK_TAG_RE = /<\s*\/?\s*(?:(?:antml:)?(?:think(?:ing)?|thought)|antthinking|final)\b/i;
const FINAL_TAG_RE = /<\s*\/?\s*final\b[^<>]*>/gi;
const THINKING_TAG_RE =
  /<\s*(\/?)s*(?:(?:antml:)?(?:think(?:ing)?|thought)|antthinking)\b[^<>]*>/gi;

function applyTrim(value: string, mode: ReasoningTagTrim): string {
  if (mode === "none") return value;
  if (mode === "start") return value.trimStart();
  return value.trim();
}

export function stripReasoningTagsFromText(
  text: string,
  options?: { mode?: ReasoningTagMode; trim?: ReasoningTagTrim },
): string {
  if (!text) return text;
  if (!QUICK_TAG_RE.test(text)) return text;

  const mode = options?.mode ?? "strict";
  const trimMode = options?.trim ?? "both";

  let cleaned = text;
  if (FINAL_TAG_RE.test(cleaned)) {
    FINAL_TAG_RE.lastIndex = 0;
    const finalMatches: Array<{ start: number; length: number; inCode: boolean }> = [];
    const preCodeRegions = findCodeRegions(cleaned);
    for (const match of cleaned.matchAll(FINAL_TAG_RE)) {
      const start = match.index ?? 0;
      finalMatches.push({ start, length: match[0].length, inCode: isInsideCode(start, preCodeRegions) });
    }
    for (let i = finalMatches.length - 1; i >= 0; i--) {
      const m = finalMatches[i];
      if (!m.inCode) {
        cleaned = cleaned.slice(0, m.start) + cleaned.slice(m.start + m.length);
      }
    }
  } else {
    FINAL_TAG_RE.lastIndex = 0;
  }

  const codeRegions = findCodeRegions(cleaned);
  THINKING_TAG_RE.lastIndex = 0;
  let result = "";
  let lastIndex = 0;
  let inThinking = false;

  for (const match of cleaned.matchAll(THINKING_TAG_RE)) {
    const idx = match.index ?? 0;
    const isClose = match[1] === "/";
    if (isInsideCode(idx, codeRegions)) continue;
    if (!inThinking) {
      result += cleaned.slice(lastIndex, idx);
      if (!isClose) inThinking = true;
    } else if (isClose) {
      inThinking = false;
    }
    lastIndex = idx + match[0].length;
  }

  if (!inThinking || mode === "preserve") {
    result += cleaned.slice(lastIndex);
  }

  return applyTrim(result, trimMode);
}
```

### なぜこの方式が優れているか

| 手書き正規表現案 | OpenClaw コピー案 |
|---|---|
| Gemini `&#94;>thought` タグの終端パターンが不明 | OpenClaw が全プロバイダで検証済み |
| コードブロック内の `<think>` を誤除去するリスク | `findCodeRegions` + `isInsideCode` で安全にスキップ |
| 新しいタグ形式が追加されるたびに手動対応 | OpenClaw のアップデートからコピーすれば追従可能 |
