# v0.3.5 修正プラン: D1 Consolidation + Surprise Topics + YAMLフォーマット

**Date:** 2026-04-06
**Severity:** MEDIUM
**Target Version:** v0.3.5

## 1. 問題の概要

### 問題1: D1タイトルが異常に長い
現在の実装 (`consolidation.go` L156) では、D1の `title` が以下のように生成されている:
```go
Title: "Semantic Consolidation: " + strings.ReplaceAll(d1Slug, "-", " ")
```
`d1Slug` はLLMが生成したURLセーフな識別子にタイムスタンプを付加したもの。これをそのままスペース区切りでタイトルにしているため、**100文字以上の意味不明なタイトル**が生成される。

### 問題2: D1要約が内容の言語を無視して英語で生成される
`buildConsolidationPrompt()` のプロンプトが**英語固定**であるため、日本語の会話から生成されたD0エピソードでも、D1要約が英語で出力される。

### 問題3: D1エピソードにトピックが正しく出力されない
現在、D1エピソードの `topics` フィールドは子D0エピソードのトピックを継承するだけ（`aggregateClusterTopics()`）。子D0にトピックがない場合、D1のtopicsは空になり、`omitempty` によりYAMLから省略される。

### 問題4: Surpriseトリガーのエピソードに `topics` がない
Surpriseトリガー時（`ai.ingest` / `ai.batchIngest`）、`params.Topics` が空、`params.Tags` がシステムタグ（`auto-segmented`, `surprise-boundary` など）のみ。`topicsForRecord()` → `LegacyTopicsFromTags()` がシステムタグを除外 → 結果 `nil` → `omitempty` によりYAMLから省略される。

### 問題5: YAMLフォーマットが壊れている可能性
D1エピソードのYAML出力で `tags:` の後に空行がある、またはリスト形式が `* item` になっている（`yaml.v3` エンコーダのデフォルトは `- item`）。

## 2. 修正内容

### 2.1. D1タイトルをLLMで生成（言語一致 + 短縮）

**現在:**
```go
Title: "Semantic Consolidation: " + strings.ReplaceAll(d1Slug, "-", " ")
```

**修正後:**
D1本文（`d1Body`）からLLMに短タイトルを生成させる。

```go
// D1本文から短タイトルを生成（最大8単語、内容と同じ言語）
titlePrompt := fmt.Sprintf(`Generate a very short title (max 8 words) for this memory summary.
Use the same language as the content below.
Return ONLY the title text, nothing else.

%s`, d1Body[:min(len(d1Body), 500)])
titleStr, titleErr := llm.GenerateText(ctx, titlePrompt)
if titleErr != nil || strings.TrimSpace(titleStr) == "" {
    truncated := strings.TrimSpace(d1Body)
    if len(truncated) > 60 {
        truncated = truncated[:60] + "..."
    }
    titleStr = truncated
}
d1Title := strings.TrimSpace(titleStr)
d1Title = strings.ReplaceAll(d1Title, "\n", " ")
d1Title = strings.ReplaceAll(d1Title, "\r", " ")
if len([]rune(d1Title)) > 120 {
    runes := []rune(d1Title)
    d1Title = string(runes[:120]) + "..."
}
```

### 2.2. D1要約プロンプトに言語一致指示を追加

両方のプロンプト（シングルトン・複数D0）に以下を追加:
```
IMPORTANT: Write the summary in the SAME LANGUAGE as the original content. If the content is in Japanese, write in Japanese. If in English, write in English. Do not translate or mix languages.
```

### 2.3. D1トピックをLLMで生成

D1本文生成後、LLMにトピックを生成させる。

```go
topicsPrompt := fmt.Sprintf(`Extract 3-5 topic keywords from this memory summary.
Use the same language as the content.
Return ONLY a comma-separated list of topics, nothing else.

%s`, d1Body[:min(len(d1Body), 500)])
topicsStr, topicsErr := llm.GenerateText(ctx, topicsPrompt)
var d1Topics []string
if topicsErr == nil && strings.TrimSpace(topicsStr) != "" {
    for _, t := range strings.Split(topicsStr, ",") {
        t = strings.TrimSpace(t)
        if t != "" {
            d1Topics = append(d1Topics, t)
        }
    }
    d1Topics, _ = ValidateTopics(d1Topics)
}
if len(d1Topics) == 0 {
    d1Topics = aggregateClusterTopics(selectedRecords)
}
if len(d1Topics) == 0 {
    d1Topics = []string{"memory"}
}
```

### 2.4. Topics フィールドのYAML出力保証（Surprise + D1 両方対応）

**ファイル:** `go/frontmatter/frontmatter.go`

`Serialize()` 関数で `Topics` が `nil` の場合に空配列を設定し、常に `topics:` フィールドが出力されるようにする。

```go
// Serialize() 内、L87 の直前に追加:
if doc.Metadata.Topics == nil {
    doc.Metadata.Topics = []string{}
}
```

**または** `omitempty` を削除:
```go
Topics []string `yaml:"topics"`
```

### 2.5. Surpriseエピソードのトピック生成（オプション）

Surpriseトリガー時にLLMでトピックを生成するのはコストが高い（追加API呼び出し）。代わりに、`params.Tags` からシステムタグを除外した残りをトピックとして使用する既存ロジックを維持し、`Topics` フィールドが常に出力されるようにする（2.4で対応）。

将来的には、D1同様にLLMでトピックを生成するオプションを追加可能。

### 2.6. YAMLフォーマットの修正

`yaml.v3` エンコーダのデフォルトスタイルを確認。`- item` 形式が正しく出力されるように、エンコーダの設定を確認する。

```go
encoder := yaml.NewEncoder(&buf)
encoder.SetIndent(2)
// 必要に応じてスタイルを設定
```
