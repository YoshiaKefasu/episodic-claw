# v0.3.7.2 改善プラン: エピソード出力ディレクトリ分離（notes / dream）

**Date:** 2026-04-08
**Severity:** LOW-MEDIUM (運用上の整理。機能に影響なし)
**Target Version:** v0.3.7.2

## 1. 問題の概要

### 現状: 全エピソードタイプが同一ディレクトリに混在

現在、3種類のエピソードがすべて `episodes/YYYY/MM/DD/` に書き出される:

| タイプ | 発生元 | 現在パス | 識別タグ |
|---|---|---|---|
| D0（自動セグメント） | `ai.batchIngest` | `episodes/YYYY/MM/DD/` | `auto-segmented`, `chunked` |
| D0（手動 ep-save） | `ai.ingest` | `episodes/YYYY/MM/DD/` | `manual-save` |
| D1（Sleep Consolidation） | `RunConsolidation` | `episodes/YYYY/MM/DD/` | `d1-summary` |

**問題点:**
1. **運用上の可読性低下**: 手動メモ、自動セグメント、D1要約が同じフォルダに混在
2. **ファイル管理の困難性**: エピソード数が増えると、どのファイルが手動メモでD1要約か区別がつかない
3. **バックアップ/エクスポートの粒度調整が不可能**: D1のみバックアップ、notesのみ共有、などの操作ができない

### 目標パス構造

```
episodes/
├── YYYY/
│   └── MM/
│       └── DD/
│           └── episode-{md5}.md          ← D0（自動セグメント）
├── notes/
│   └── YYYY-MM/
│       └── episode-{md5}.md              ← D0（手動 ep-save）
├── dream/
│   └── YYYY-MM/
│       └── DD/
│           └── {d1-slug}.md              ← D1（Sleep Consolidation）
└── vector.db                              ← Pebble DB（変更なし）
```

## 2. 修正内容

### 2.1 アプローチ: Option A — インライン修正 (KISS準拠)

新しい抽象化層（PathBuilder等）は作成しない。YAGNI違反になるため、既存のパス生成ロジックに分岐を追加するインライン修正を採用。

### 2.2 影響範囲分析

**変更が必要なファイル:**

| ファイル | 関数 | 変更内容 |
|---|---|---|
| `go/main.go` | `handleIngest()` | `tags` に `manual-save` が含まれる場合、`notes/YYYY-MM/` に出力 |
| `go/main.go` | `handleBatchIngest()` | D0自動セグメントは既存パス維持（コメント追加のみ） |
| `go/internal/vector/consolidation.go` | `processCluster()` | D1を `dream/YYYY-MM/DD/` に出力 |

**スキャンパターン拡張（重要）:**

| ファイル | 関数 | 変更内容 |
|---|---|---|
| `go/internal/vector/d1_clustering.go` | `collectActiveD0Nodes()` | `notes/` ディレクトリも探索対象に追加 |
| `go/internal/vector/background.go` | `processBacklogFile()` | `dream/` ディレクトリも探索対象に追加 |

### 2.3 パス設計

**handleIngest (ep-save 判定ロジック):**
```go
isManualSave := false
for _, t := range params.Tags {
    if t == "manual-save" {
        isManualSave = true
        break
    }
}

var dirPath string
if isManualSave {
    dirPath = filepath.Join(params.AgentWs, "notes",
        fmt.Sprintf("%04d-%02d", now.Year(), now.Month()))
} else {
    dirPath = filepath.Join(params.AgentWs,
        fmt.Sprintf("%04d", now.Year()),
        fmt.Sprintf("%02d", now.Month()),
        fmt.Sprintf("%02d", now.Day()))
}
```

**handleBatchIngest (D0 自動セグメント):**
```go
// 変更なし — 既存 YYYY/MM/DD/ パスを維持（D0自動セグメント用）
dirPath := filepath.Join(params.AgentWs,
    fmt.Sprintf("%04d", now.Year()),
    fmt.Sprintf("%02d", now.Month()),
    fmt.Sprintf("%02d", now.Day()))
```

**processCluster (D1 Consolidation):**
```go
// 変更: dream/YYYY-MM/DD/
dirPath := filepath.Join(agentWs, "dream",
    fmt.Sprintf("%04d-%02d", now.Year(), now.Month()),
    fmt.Sprintf("%02d", now.Day()))
```

### 2.4 HealingWorker のスキャンパターン拡張

**方針:** ディレクトリパターンをハードコードではなく、スキャン関数に引数として渡す形にリファクタリング。

`collectActiveD0Nodes()` が現在 `filepath.Walk` で全サブディレクトリを再帰探索している場合、`notes/` は自動的に拾われる（Walkは再帰的）。ただし `dream/` 内のD1ファイルは対象外とするため、`tags: ["d1-summary"]` でフィルタリングする。

`processBacklogFile()` も同様に再帰Walkを使用している場合、新ディレクトリは自動的に拾われる。

**確認が必要な点:**
- `collectActiveD0Nodes()` が特定のディレクトリパターンに限定されていないか
- `tags: ["d1-summary"]` のエピソードがD1として正しく除外されるか

### 2.5 下位互換性

**既存エピソードファイル:** 既存の `YYYY/MM/DD/` 配下のファイルはそのまま。HealingWorker は全パターンをスキャンする。

**Pebble DB:** ファイルパスは `SourcePath` として記録されるが、recall/search はベクトル検索が主。パス変更はDBに影響しない。

**ep-expand / ep-recall:** slugベースで検索するため、ディレクトリ構成の変更は透過的。
