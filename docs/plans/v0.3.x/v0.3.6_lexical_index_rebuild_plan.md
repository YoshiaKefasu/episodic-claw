## 2. 修正内容

### 2.1. `RebuildLexicalIndex()` メソッドの追加

**ファイル:** `go/internal/vector/store.go`（新規メソッド）

PebbleDB内の全 `ep:` レコードをイテレートし、それぞれを `sys_lexq:` キューにエンキューする。Lexicalワーカーがバックグラウンドで順次処理するため、ブロッキングなし。

**修正済みコード（Round 1/2 Audit 反映済）:**

```go
// RebuildLexicalIndex scans all EpisodeRecord entries in PebbleDB and
// re-enqueues them into the sys_lexq queue for the lexical worker to
// index into Bleve. Safe to call on a live store — the worker processes
// items asynchronously.
func (s *Store) RebuildLexicalIndex() (int, error) {
	if s.lexical == nil {
		return 0, fmt.Errorf("lexical index not initialized")
	}

	// Scope lock narrowly — follow CleanOrphans pattern (store.go:272-277)
	s.mutex.RLock()
	iter, err := s.db.NewIter(&pebble.IterOptions{
		LowerBound: prefixEp,
		UpperBound: []byte("ep;"),
	})
	s.mutex.RUnlock()
	if err != nil {
		return 0, err
	}
	defer iter.Close()

	batch := s.db.NewBatch()
	defer batch.Close()

	count := 0
	for iter.First(); iter.Valid(); iter.Next() {
		var rec EpisodeRecord
		if err := msgpack.Unmarshal(iter.Value(), &rec); err != nil {
			continue
		}
		if rec.SourcePath == "" {
			continue
		}
		if _, err := os.Stat(rec.SourcePath); os.IsNotExist(err) {
			continue
		}
		s.enqueueSysLexq(batch, "UPDATE", rec.ID)
		count++
	}

	if err := batch.Commit(pebble.Sync); err != nil {
		return 0, fmt.Errorf("failed to commit lexical rebuild batch: %w", err)
	}
	return count, nil
}
```

### 2.4. HealingWorker Pass 5: Smart Lexical Consistency Check

**ファイル:** `go/main.go` `RunAsyncHealingWorker()` 内

既存のPass 1-4の直後に、Lexicalインデックスの健全性を定期的にチェックするPass 5を追加する。

**設計方針:**
1.  **チェック間隔**: デフォルト7日間（1週間）。
2.  **即時チェック**: 前回のチェック記録（タイムスタンプ）が存在しない場合は、初回起動時に即時チェックを実行。
3.  **欠落のみ補充**: `sys_lexq` キューの `"UPDATE"` アクションを利用し、既存のインデックスは上書き（冪等）、欠落しているレコードのみを補充する。
4.  **設定反映**: `openclaw.plugin.json` の `lexicalRebuildIntervalDays` を尊重する。

**実装詳細:**

```go
// Pass 5: Lexical Consistency Check (runs every HealingWorker cycle)
// Checks if >10% of records are missing from the lexical index.
func (s *Store) CheckAndRebuildLexicalIfNeeded(intervalDays int) {
    // 1. Throttle: Check only once per interval
    lastCheck, _ := s.db.Get([]byte("meta:last_lexical_check"))
    if lastCheck != nil {
        ts, _ := strconv.ParseInt(string(lastCheck), 10, 64)
        if time.Now().Unix()-ts < int64(intervalDays)*86400 {
            return // Too soon
        }
    }

    // 2. Check health: If >10% missing, trigger rebuild
    total := s.Count()
    lexical, _ := s.LexicalCount()
    
    // s.Count() returns maxID, so we use a conservative threshold
    if total > 0 && lexical < uint64(float64(total)*0.9) {
        // Prevent concurrent rebuilds (atomic guard needed in Store struct)
        if !s.rebuildInProgress.CompareAndSwap(false, true) {
            return // Already running
        }
        defer s.rebuildInProgress.Store(false)

        EmitLog("HealingWorker: [Pass 5] Lexical gap detected (%d/%d). Triggering rebuild.", lexical, total)
        count, err := s.RebuildLexicalIndex()
        if err != nil {
            EmitLog("HealingWorker: [Pass 5] Rebuild failed: %v", err)
        } else {
            EmitLog("HealingWorker: [Pass 5] Rebuild complete. Enqueued %d records.", count)
        }
    }

    // 3. Update timestamp in PebbleDB metadata
    batch := s.db.NewBatch()
    batch.Set([]byte("meta:last_lexical_check"), []byte(fmt.Sprintf("%d", time.Now().Unix())), nil)
    batch.Commit(pebble.Sync)
    batch.Close()
}
```

### 📍 Pass 5 Integration Assessment

**Insertion point**: ✅ Correct. `RunAsyncHealingWorker()` ends at main.go:1806 immediately after Pass 4 (GC). Pass 5 code would insert cleanly between line 1805 (`}`) and line 1806 (`}`).

**`s.lastLexicalCheck` field**: 🔴 **Does not exist on Store struct.** The Store struct (store.go:125-136) has no `lastLexicalCheck time.Time` field. Adding it requires:
- Modifying the Store struct definition (store.go:136)
- Initializing it in `NewStore()` (store.go:186-195)
- This is a **breaking change** to the struct — any code that constructs Store literals would break (none currently does, but it's a structural change)
- **Solution in Plan**: Use PebbleDB metadata key `meta:last_lexical_check` instead of struct field. **No struct change needed.**

**`cfg.lexicalRebuildIntervalDays`**: 🔴 **No config path exists.** `RunAsyncHealingWorker(agentWs, apiKey, vstore)` at main.go:1494 receives **no config parameter**. 
- **Solution in Plan**: Pass interval via CLI flag (`--lexical-rebuild-interval`) from TypeScript spawn. Read via `flag.Int()` in `main.go`.

**Thread safety of `s.lastLexicalCheck`**: ⚠️ The HealingWorker runs in a single goroutine per workspace (main.go:123-138), so within one worker there's no race. However, `RebuildLexicalIndex()` is also callable via the RPC handler (Step 3), which runs in a separate goroutine (`go handleConnection`). If the RPC triggers a rebuild while Pass 5 is mid-check, `lastLexicalCheck` could be stale. **No mutex protects this field.**
- **Solution in Plan**: Use `atomic.Bool` (`rebuildInProgress`) in Store struct to prevent concurrent rebuilds.
