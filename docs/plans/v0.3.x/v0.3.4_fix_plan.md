# v0.3.4 実装プラン: assemble() 停止への対応とセグメンテーション復活

## 1. 問題の概要

OpenClawのコミット `235908c30e`（2026-03-30）によるマルチキンドプラグインのスロット所有権ロジック変更により、episodic-clawの `assemble()` が呼ばれなくなった。これにより以下の機能が完全に停止している：

- **自動エピソード生成**: `segmenter.processTurn()` → `segmentScore()` → `batchIngest()` のチェーンが不発
- **メモリ注入**: `prependSystemContext` によるエピソードメモリのシステムプロンプト注入が不発

**影響期間:** 2026-04-01 20:38 以降（4日以上）
**根本原因:** OpenClawは `config.plugins.slots.contextEngine` で指定された**1つの**コンテキストエンジンのみ `assemble()` を呼ぶ。未設定時は `"legacy"` にフォールバックし、episodic-clawは無視される。

---

## 2. 修正アプローチ

### 2.1. アーキテクチャ変更: `before_prompt_build` をメイントリガーに

OpenClawの `before_prompt_build` フックは**コンテキストエンジンのスロット設定に関係なく**毎ターン呼ばれる（`attempt.ts` L1469-1494）。これをセグメンテーションとメモリ注入の新しいトリガーにする。

**OpenClawの実行フロー:**
```
run attempt
  -> assemble()              ← 呼ばれない（contextEngine slotが"legacy"）
  -> before_prompt_build     ← 毎ターン確実に呼ばれる ★ここを使う
  -> prompt -> model
```

**`before_prompt_build` フックの仕様（OpenClawソース確認済み）:**
- **event:** `{ prompt: string, messages: unknown[] }`
- **context:** `{ runId, agentId, sessionKey, sessionId, workspaceDir, messageProvider, trigger, channelId }`
- **戻り値:** `{ prependSystemContext?, appendSystemContext?, prependContext?, systemPrompt? }`
- **制限:** `tokenBudget` は渡されない（文字数ベースのトークン見積もりが必要）

### 2.2. 既存機能との共存

| 機能 | 現在のトリガー | v0.3.4のトリガー | 変更内容 |
|------|---------------|-----------------|---------|
| セグメンテーション | `assemble()` 内 | `before_prompt_build` 内 | 移行 |
| メモリ注入（recall） | `assemble()` 内 | `before_prompt_build` 内 | 移行 |
| `ep-recall` ツール | 手動呼び出し | 変更なし | そのまま |
| `ep-save` ツール | 手動呼び出し | 変更なし | そのまま |
| `ep-expand` ツール | 手動呼び出し | 変更なし | そのまま |
| `ep-anchor` ツール | 手動呼び出し | 変更なし | そのまま |
| `assemble()` | OpenClaw呼び出し | ダイアグノスティック logging のみ | 縮小 |

## 3. 実装詳細

### 3.1. `before_prompt_build` フックの登録

**挿入位置:** `src/index.ts` の `after_compaction` フック登録ブロックの直後（L674の次行）、`registerContextEngine` の前。

```typescript
// ─── before_prompt_build フック: セグメンテーション + メモリ注入 ───
// assemble() が呼ばれない環境でのフォールバック。
// OpenClawの contextEngine slot 設定に関係なく毎ターン確実に呼ばれる。
api.on("before_prompt_build", async (event: any, ctx: any) => {
  const msgs = ((event && event.messages) || []) as Message[];
  const resolution = resolveAgentWorkspaces(ctx, openClawGlobalConfig);
  const { agentId, agentWs } = resolution;
  if (!agentWs) return {};

  await prepareWorkspaces(resolution);
  const state = getAgentState(agentId);
  state.lastAgentWs = agentWs;

  // ── セグメンテーション（fire-and-forget）──
  state.segmenter.processTurn(msgs, agentWs, agentId).catch(err => {
    console.log("[Episodic Memory] Fallback segmenter error in before_prompt_build:", err);
  });

  // ── メモリ注入（tokenBudget なし → 文字数見積もり）──
  const reserveTokens = cfg.reserveTokens ?? 2048;
  // tokenBudget が取れないため、固定上限で安全側に倒す
  const ESTIMATED_MAX_EPISODIC_TOKENS = 1024;
  const k = 5;
  const recentMessages = msgs.slice(-5);
  const queryParts = recentMessages
    .map(m => {
      const content = extractText(m.content).trim();
      return content ? `${m.role}: ${content}` : "";
    })
    .filter(part => part.length > 0);
  const fullQuery = queryParts.join("\n").trim();

  if (!fullQuery) return {};

  try {
    const result = await state.retriever.recall({
      agentWs,
      agentId,
      query: fullQuery,
      k,
      minScore: 0.0,
      strictTopics: false,
      topics: [],
      maxChars: ESTIMATED_MAX_EPISODIC_TOKENS * 4, // 粗略: 1 token ≈ 4 chars
    });

    if (!result || result.episodes.length === 0) return {};

    // エピソードテキストを構築
    const episodeTexts = result.episodes
      .map((ep: any) => ep.text || "")
      .filter(Boolean)
      .join("\n\n");

    if (!episodeTexts) return {};

    return { prependSystemContext: episodeTexts };
  } catch (err) {
    console.log("[Episodic Memory] before_prompt_build recall error:", err);
    return {};
  }
});
```

### 3.2. `assemble()` の縮小（ダイアグノスティックのみ）

既存の `assemble()` は残すが、セグメンテーションとメモリ注入の処理を削除し、診断ログのみとする。`before_prompt_build` が正常に動いている場合、`assemble()` は呼ばれないためこのコードは実行されないが、万一呼ばれた場合の保険として最小限の機能を残す。

```typescript
async assemble(ctx: any) {
  const msgs = (ctx.messages || []) as Message[];
  console.log(`[Episodic Memory] assemble() called (${msgs.length} messages) — this means contextEngine slot IS set to "episodic-claw"`);

  // before_prompt_build が既にセグメンテーション+メモリ注入を担当しているため、
  // assemble() では最小限の処理のみ行う（二重注入防止）
  return { messages: msgs, prependSystemContext: "", estimatedTokens: 0 };
}
```

## 5. 既知の制限事項とトレードオフ

### 5.1. トークン予算の非対称性

`before_prompt_build` フックは `tokenBudget` を受け取らない。そのため、メモリ注入のサイズを `assemble()` のように正確に制御できない。

**対策:** 固定上限 `ESTIMATED_MAX_EPISODIC_TOKENS = 1024` で安全側に倒す。`maxChars = 1024 * 4 = 4096` 文字でrecall。

**トレードオフ:**
- Pros: `tokenBudget` に依存せず確実にメモリ注入できる
- Cons: コンテキストに余裕がある場合でも1024トークンまでに制限される

### 5.2. 二重注入の防止

`before_prompt_build` と `assemble()` の両方が呼ばれる環境（`contextEngine` slot が `"episodic-claw"` に設定されている場合）では、メモリが二重注入される可能性がある。

**対策:** `assemble()` を最小限（空の `prependSystemContext` を返す）に縮小。`before_prompt_build` が既に注入済みなので、`assemble()` での再注入は不要。

### 5.3. `allowPromptInjection` 設定

OpenClawの `plugins.entries.episodic-claw.hooks.allowPromptInjection` が `false` に設定されている場合、`before_prompt_build` フックはブロックされる。

**対策:** デフォルトで `true`（設定しない場合は許可）。ユーザーが明示的に `false` にしない限り問題ない。
