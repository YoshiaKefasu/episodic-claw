# v0.3.5 Markdown Format Improvement 実装プラン

**Date:** 2026-04-06
**Severity:** MEDIUM
**Target Version:** v0.3.5

## 1. 問題の概要

### 問題: エピソードMarkdown本文にTelegramの生JSONメタデータが混入

各エピソードファイルの本文に、以下のJSONブロックがそのまま含まれている：
- `Conversation info (untrusted metadata):` — `message_id`, `reply_to_id`, `sender_id`, `sender`, `timestamp`, `has_reply_context`
- `Sender (untrusted metadata):` — `label`, `id`, `name`
- `Replied message (untrusted, for context):` — `sender_label`, `body`

**影響:**
- ファイルあたり約3000トークンの無駄な消費
- LLMのコンテキスト枠を圧迫
- 人間にとっての可読性・検索性（Grep）が著しく低下
- `ep-recall` 実行時のノイズ増加

---

## 2. 根本原因

JSONメタデータはepisodic-clawではなく、**OpenClawのTelegramゲートウェイ**がメッセージ内容に注入している。episodic-clawの全パイプラインでクリーニングが行われていない：

| ステージ | ファイル | クリーニング |
|---------|---------|-------------|
| メッセージ受信 | `src/index.ts` | なし |
| セグメンターバッファ | `src/segmenter.ts` | ロールフィルタのみ |
| テキスト抽出 | `src/large-payload.ts` | なし（そのまま抽出） |
| サマリー構築 | `src/summary-escalation.ts` | なし（そのまま連結） |
| Go batch ingest | `go/main.go` | なし（SummaryをそのままBodyに） |
| Frontmatter serialize | `go/frontmatter/frontmatter.go` | なし |
| バックグラウンドインデクサー | `go/internal/vector/background.go` | なし |

## 3. 修正内容

### 3.1. テキスト抽出時にTelegram JSONメタデータを除去（主修正）

**ファイル:** `src/large-payload.ts`
**関数:** `extractPlainText()` または `normalizeMessageText()`

Telegramメタデータパターンを正規表現で除去する。

```typescript
// Telegram metadata patterns to strip from message content
const TELEGRAM_META_PATTERNS = [
  // Conversation info (untrusted metadata): ... ```json ... ```
  /Conversation info \(untrusted metadata\):[\s\S]*?```json[\s\S]*?```/gi,
  // Sender (untrusted metadata): ... ```json ... ```
  /Sender \(untrusted metadata\):[\s\S]*?```json[\s\S]*?```/gi,
  // Replied message (untrusted, for context): ... ```json ... ```
  /Replied message \(untrusted,? for context\):[\s\S]*?```json[\s\S]*?```/gi,
  // Fallback: any standalone ```json blocks with metadata headers
  /\(untrusted metadata\):[\s\S]*?```json[\s\S]*?```/gi,
];

function stripTelegramMetadata(text: string): string {
  let cleaned = text;
  for (const pattern of TELEGRAM_META_PATTERNS) {
    cleaned = cleaned.replace(pattern, "").trim();
  }
  return cleaned;
}
```

`normalizeMessageText()` で `stripToolOutputPatterns()` の後に適用：

```typescript
export function normalizeMessageText(content: unknown): string {
  const text = extractPlainText(content);
  // Strip tool output artifacts
  const stripped = stripToolOutputPatterns(text);
  // Strip Telegram JSON metadata blocks
  return stripTelegramMetadata(stripped);
}
```

### 3.2. サマリー構築時の空行除去（副修正）

**ファイル:** `src/summary-escalation.ts`
**関数:** `buildNormalSummary()`, `buildAggressiveSummary()`, `buildFallbackSummary()`

JSON除去後に空の `role: ` 行が残る可能性があるため、フィルタリングを強化：

```typescript
// 既存のフィルタに空行チェックを追加
.filter((line) => line.length > 0 && line.trim() !== "user: " && line.trim() !== "assistant: ")
```

### 3.3. Goサイドカーのサニタイズ（安全網）

**ファイル:** `go/main.go`
**関数:** `handleBatchIngest()`, `handleIngest()`

TypeScript側のクリーニングが何らかの理由で漏れた場合の最終防御：

```go
// stripTelegramMetadata removes Telegram JSON metadata blocks from text
func stripTelegramMetadata(text string) string {
    patterns := []*regexp.Regexp{
        regexp.MustCompile(`(?i)Conversation info \(untrusted metadata\):[\s\S]*?` + "```json[\\s\\S]*?```"),
        regexp.MustCompile(`(?i)Sender \(untrusted metadata\):[\s\S]*?` + "```json[\\s\\S]*?```"),
        regexp.MustCompile(`(?i)Replied message \(untrusted,? for context\):[\s\S]*?` + "```json[\\s\\S]*?```"),
    }
    cleaned := text
    for _, p := range patterns {
        cleaned = p.ReplaceAllString(cleaned, "")
    }
    return strings.TrimSpace(cleaned)
}
```

`handleBatchIngest()` で `it.Summary` をシリアライズ前にクリーニング：

```go
// Sanitize summary before writing
it.Summary = stripTelegramMetadata(it.Summary)
```

### 3.4. Lazy Migration on Healing（既存ファイル対応）

**ファイル:** `go/internal/vector/background.go`
**関数:** `processBacklogFile()`, `RunAsyncHealingWorker()`

既存の汚染ファイルをHealingWorkerが再インデックスする際に、JSONメタデータを除去してファイルを上書き保存する。

```go
// In processBacklogFile() or healing pass:
if strings.Contains(body, "untrusted metadata") {
    cleanedBody := stripTelegramMetadata(body)
    if cleanedBody != body {
        // Rewrite file with cleaned body
        doc := &frontmatter.MarkdownDocument{
            Metadata: doc.Metadata,
            Body:     cleanedBody,
        }
        frontmatter.Serialize(filePath, doc)
    }
}
```
