# 実装プラン: v0.4.15 — `openrouterConfig.maxTokens` 伝達パス欠落（リグレッション）

---

## 0. このドキュメントの位置づけ

- 目的: v0.4.14 Fix B で `narrativeMaxTokens` を削除した際に、`openrouterConfig.maxTokens` を OpenRouterClient に伝達する唯一のコードパスも消えたリグレッションを修正する
- 深刻度: **CRITICAL** — ユーザーが `openrouterConfig.maxTokens: 4096` を設定しても OpenRouter API に `max_tokens` が送信されず、モデルのデフォルト最大出力長が使われる
- 発見元: `docs/incidents/v0.4.14_config_pipeline_regression_audit.md` セクション1
- ソースコード検証: **済**

## 1. 問題定義

### データフロー（v0.4.13 → 削除前）

```
openclaw.json: openrouterConfig: { maxTokens: 4096 }
  → rawConfig.openrouterConfig.maxTokens = 4096
    → loadConfig() L100: rawConfig?.openrouterConfig?.maxTokens ?? rawConfig?.narrativeMaxTokens
    → cfg.narrativeMaxTokens = 4096
      → index.ts L507: maxTokens: cfg.narrativeMaxTokens
        → OpenRouterClient.config.maxTokens = 4096
          → body.max_tokens = 4096 ✅
```

### データフロー（v0.4.14 → 削除後）

```
openclaw.json: openrouterConfig: { maxTokens: 4096 }
  → rawConfig.openrouterConfig.maxTokens = 4096
    → loadConfig(): narrativeMaxTokens 行が存在しない ❌
    → cfg に maxTokens を伝達するフィールドが存在しない ❌
      → index.ts L499: maxTokens パラメータなし
        → OpenRouterClient.config.maxTokens = undefined
          → body.max_tokens は送信されない ❌
```

### 証拠

| # | ファイル | 行 | 内容 |
|---|----------|-----|------|
| 1 | `config.ts` | L67-112 | `narrativeMaxTokens` 行が存在せず、`openrouterConfig.maxTokens` を平坦化する行もない |
| 2 | `index.ts` | L498-504 | `OpenRouterClient` 生成に `maxTokens` パラメータがない |

## 2. 修正内容

### 2.1 `config.ts` — 新しい平坦化フィールドを追加

```typescript
// L102 の直後（openrouterModel の後）に追加:
openrouterModel: rawConfig?.openrouterConfig?.model ?? rawConfig?.openrouterModel ?? "openrouter/free",
openrouterMaxTokens: rawConfig?.openrouterConfig?.maxTokens,
narrativeTemperature: ...
```

フィールド名は `openrouterMaxTokens`。元の `narrativeMaxTokens` ではなく、出所（`openrouterConfig`）を明示する名前にする。`rawConfig?.narrativeMaxTokens` へのフォールバックは含めない（スキーマの `additionalProperties: false` で弾かれるため常に `undefined`）。

### 2.2 `types.ts` — 新しいフィールドを追加

```typescript
/** Max tokens cap for narrative generation. Sources from openrouterConfig.maxTokens.
 *  Omit to use the model's default context window. */
openrouterMaxTokens?: number;
```

`narrativeMaxTokens` の削除位置（元 L122-125 の場所）に配置。

### 2.3 `index.ts` — OpenRouterClient に maxTokens を渡す

```typescript
// Before:
new OpenRouterClient({
  apiKey: cfg.openrouterApiKey,
  model: cfg.openrouterModel,
  temperature: cfg.narrativeTemperature,
  reasoning: cfg.openrouterReasoning,
})

// After:
new OpenRouterClient({
  apiKey: cfg.openrouterApiKey,
  model: cfg.openrouterModel,
  maxTokens: cfg.openrouterMaxTokens,
  temperature: cfg.narrativeTemperature,
  reasoning: cfg.openrouterReasoning,
})
```

`OpenRouterConfig.maxTokens` は `optional` なので、`cfg.openrouterMaxTokens` が `undefined` の場合は送信されず、OpenRouter がモデルデフォルトを使う（推奨動作）。
