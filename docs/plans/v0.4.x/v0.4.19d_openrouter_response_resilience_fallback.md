# 実装プラン: v0.4.19d — OpenRouter Response Resilience & Fallback Overhaul

> **深刻度: 🔴 HIGH** — openrouter/free の不安定な上流レスポンスがエピソード紛失・品質低下の主因

---

## 0. このドキュメントの位置づけ

- **目的**: OpenRouter freeティアの上流応答揺れ（HTTP 200でerror包み、空content、プロバイダポリシー拒否）に対する包括的レジリエンス強化
- **前提**: v0.4.17（Narrative Output Control Overhaul）以降、品質ゲートは5ゲート体制で稼働中。本プランは「LLM呼び出し自体の耐障害性」に焦点を当てる
- **発見元**: 実テストでのOpenRouter空レスポンス発生（証拠1）、HTTP 200で `{"error":{"message":"Provider returned error","code":503}}` 返却の確認（証拠2）、プロバイダ側モデル切替えによる400エラー再現（証拠3）
- **変更ファイル（実装対象）**: `openrouter-client.ts`, `narrative-worker.ts`

---

## 1. 問題定義

### 1.1 根本原因: OpenRouter freeティアの3つの応答揺れパターン

| # | パターン | HTTP | レスポンス body | 現状の挙動 | 影響 |
|---|---------|------|----------------|-----------|------|
| A | **choicesなし空ボディ** | 200 | `{"choices":[]}` または `{}` | `content !== "string"` → `"OpenRouter returned empty or invalid response"` 投げ | NarrativeWorker品質ゲート「全再試行失敗」→cacheRetry |
| B | **error包み200** | 200 | `{"error":{"message":"Provider returned error","code":503}}` | 同上（`data.choices`なし） | 再試行可能な503/429が再試行されず即座に失敗 |
| C | **プロバイダポリシー400** | 200 または 400 | `{"error":{"message":"Developer instruction is not enabled for models/gemma-3n-e4b-it","code":400}}` | HTTP 400の場合は即座にスロー（再試行不可）、200包みの場合は(A)と同じ | プロバイダ切替えによる一時的エラー |

### 1.2 現状の欠陥

| 欠陥 | 詳細 |
|------|------|
| **HTTPステータスのみで再試行判定** | `doCompletion()` は `!response.ok` のみでスロー。HTTP 200でerror包まれたレスポンスは再試行ロジックに入らない |
| **空レスポンスの分類なし** | 全て `"OpenRouter returned empty or invalid response"` の1行にまとめられ、観測性ゼロ |
| **リトライ回数が少ない** | NarrativeWorker `MAX_RETRIES=5`。freeティアの揺れには不十分 |
| **バックオフが短い** | `RETRY_BASE_DELAY_MS=1000`（1秒開始）、cap30秒。プロバイダの503復旧には短すぎる |
| **フォールバックモデルなし** | カスタムモデル使用時に全再試行失敗→终结。`openrouter/free` へのフォールバックが存在しない |
| **重複保存防止なし** | 同一rawTextの再キュー→再生成→多重保存の経路が存在 |
| **サーキットブレーカなし** | openrouter/free で連続空レスポンス時の自動切り替えが存在しない |

### 1.3 影響範囲の定量評価

```
最悪ケース（現状）:
  再試行5回 × バックオフ1s開始/cap30s = 最大約2分で全再試行失敗
  → cacheRetryで再キュー → 20回試行上限で永続失敗の可能性
  → エピソード紛失（rawTextはPebbleDBに残るが、自動復旧なし）
```

```
改善後:
  再試行12回 × バックオフ3s開始/cap600s = 最大約42.75分の猶予
  + フォールバックモデル3回 ≈ 追加10分
  → プロバイダ一時障害の大半はこの間に復旧
```

## 4. Fix 1: レスポンス判定厳格化（200+error検出）

### 4.1 問題

現状の `doCompletion()` (L126-142):

```typescript
if (!response.ok) {
  // HTTP 4xx/5xx → throw (こっちはOK)
  const err = new Error(`OpenRouter HTTP ${response.status}: ...`);
  throw err;
}

const data = await response.json();
const content = data?.choices?.[0]?.message?.content;
if (typeof content !== "string" || content.trim().length === 0) {
  throw new Error("OpenRouter returned empty or invalid response");
  // ↑ 200+error包み も choices無し も全部これ → 再試行不可 & 分類なし
}
```

**欠陥**: HTTP 200なのに `data.error` があるケースを見落としている。`data.error.code` が 503/429 の場合は再試行可能だが、現状は即座に `"empty or invalid response"` として捨てられる。

### 4.2 解決策: 3段階検出

```typescript
const data = await response.json();

// Phase 1: error包み検出 (HTTP 200 + data.error)
if (data?.error && typeof data.error === "object") {
  const errorCode = data.error.code ?? 0;
  const errorMsg = data.error.message ?? "unknown error";
  throw new OpenRouterError({
    message: `OpenRouter error_wrapped_200: code=${errorCode} message=${errorMsg.slice(0, 200)}`,
    errorClass: classifyErrorCode(errorCode),
    retriable: isRetriableErrorCode(errorCode),
    providerErrorCode: typeof errorCode === "number" ? errorCode : parseInt(String(errorCode), 10),
    providerName: data.error.metadata?.provider_name,
  });
}

// Phase 2: choices構造検証
const content = data?.choices?.[0]?.message?.content;
if (typeof content !== "string" || content.trim().length === 0) {
  // choices無し / content空 / content非string を分類
  const errorClass: OpenRouterErrorClass = !Array.isArray(data?.choices) || data.choices.length === 0
    ? "missing_choices"
    : typeof data.choices[0]?.message?.content !== "string"
    ? "non_string_content"
    : "empty_content";

  throw new OpenRouterError({
    message: `OpenRouter empty_response: ${errorClass}`,
    errorClass,
    retriable: true, // 空レスポンス系は再試行対象
  });
}

// Phase 3: 正常レスポンス
return content.trim();
```

### 4.3 HTTP 4xx/5xx の分類強化

```typescript
if (!response.ok) {
  const retryAfter = response.headers.get("Retry-After");
  const errorText = await response.text().catch(() => "");
  let errorClass: OpenRouterErrorClass;
  let retriable = false;

  if (response.status === 429) {
    errorClass = "http_429";
    retriable = true;
  } else if (response.status >= 500) {
    errorClass = "http_5xx";
    retriable = true;
  } else if (response.status === 400) {
    // 400 might be retriable if it's a provider routing issue
    errorClass = "http_400";
    retriable = false; // default: non-retriable
    // Check if body contains provider policy error
    if (errorText.includes("Developer instruction") || errorText.includes("not enabled for model")) {
      errorClass = "provider_400_policy";
      retriable = false; // モデル不一致は再試行で解決しない（フォールバック対象）
    }
  } else if (response.status === 401 || response.status === 403) {
    errorClass = `http_${response.status}`;
    retriable = false;
  } else {
    errorClass = "http_unknown";
    retriable = false;
  }

  throw new OpenRouterError({
    message: `OpenRouter HTTP ${response.status} [${errorClass}]: ${errorText.slice(0, 200)}`,
    errorClass,
    retriable,
    retryAfterSeconds: retryAfter ? parseInt(retryAfter, 10) : 0,
  });
}
```

### 4.4 分類ヘルパー関数

`openrouter-client.ts` に追加:

```typescript
/** OpenRouter error classification for observability and retry logic. */
export type OpenRouterErrorClass =
  | "missing_choices"        // 200だがchoices配列なし/空
  | "empty_content"          // 200だがcontentが空文字列
  | "non_string_content"    // 200だがcontentがstring以外
  | "error_wrapped_200"     // 200だがdata.errorあり（汎用）
  | "provider_503"          // 200+error.code=503（プロバイダ一時障害）
  | "provider_429"          // 200+error.code=429（プロバイダレート制限）
  | "provider_400_policy"   // 200+error.code=400（モデル不一致ポリシー）
  | "http_429"              // HTTP 429（OpenRouterレート制限）
  | "http_5xx"              // HTTP 5xx（サーバー障害）
  | "http_400"               // HTTP 400（その他）
  | "http_401"              // HTTP 401（認証エラー）
  | "http_403"              // HTTP 403（権限エラー）
  | "timeout"               // AbortController タイムアウト
  | "network"               // ネットワークエラー
  | "circuit_open"          // CircuitBreaker遮断中
  | "http_unknown";         // その他 HTTP エラー

function classifyErrorCode(code: number | string): OpenRouterErrorClass {
  const numCode = typeof code === "string" ? parseInt(code, 10) : code;
  if (numCode === 503) return "provider_503";
  if (numCode === 429) return "provider_429";
  if (numCode === 400) return "provider_400_policy"; // 200包みの400はポリシーエラー
  return "error_wrapped_200"; // 汎用
}

function isRetriableErrorCode(code: number | string): boolean {
  const numCode = typeof code === "string" ? parseInt(code, 10) : code;
  // 503 = 一時的プロバイダ障害, 429 = レート制限 → 再試行可能
  // 400 = ポリシー/モデル不一致 → 再試行不可（フォールバック対象）
  return numCode === 503 || numCode === 429;
}

export class OpenRouterError extends Error {
  readonly openRouterErrorClass: OpenRouterErrorClass;
  readonly retriable: boolean;
  readonly retryAfterSeconds: number;
  readonly providerErrorCode?: number;
  readonly providerName?: string;

  constructor(params: {
    message: string;
    errorClass: OpenRouterErrorClass;
    retriable: boolean;
    retryAfterSeconds?: number;
    providerErrorCode?: number;
    providerName?: string;
  }) {
    super(params.message);
    this.name = "OpenRouterError";
    this.openRouterErrorClass = params.errorClass;
    this.retriable = params.retriable;
    this.retryAfterSeconds = params.retryAfterSeconds ?? 0;
    this.providerErrorCode = params.providerErrorCode;
    this.providerName = params.providerName;
  }
}
```

## 5. Fix 2: 200+error包みのリトライ対象追加

### 5.1 現状の chatCompletion() リトライロジック

```typescript
// L48-73: chatCompletion() の catch ブロック
} catch (err) {
  lastError = err instanceof Error ? err : new Error(String(err));

  // 429 → check Retry-After header and retry
  if (lastError.message.includes("429")) {
    const retryAfter = (lastError as any).retryAfterSeconds as number ?? 0;
    if (retryAfter > 0) {
      await this.sleep(retryAfter * 1000);
      continue;
    }
  }

  // Non-retryable errors (4xx except 429)
  if (lastError.message.includes("400") || lastError.message.includes("401") || lastError.message.includes("403")) {
    throw lastError;
  }
}
```

**欠陥**:
1. メッセージ文字列への `includes()` マッチングが脆弱（誤検出リスク）
2. `retriable` / `openRouterErrorClass` メタデータが伝播されていない
3. 200+error包みの再試行可能コード(503/429)が再試行されない

### 5.2 新リトライロジック

`doCompletion()` でタグ付けしたメタデータを `chatCompletion()` で活用:

```typescript
async chatCompletion(params: {
  systemPrompt: string;
  userMessage: string;
}): Promise<string> {
  const { systemPrompt, userMessage } = params;
  let lastError: Error | null = null;
  let lastErrorClass: OpenRouterErrorClass | "unknown" = "unknown";
  let retryAfterSeconds = 0;

  for (let attempt = 0; attempt <= this.config.maxRetries; attempt++) {
    if (attempt > 0) {
      const delayMs = Math.min(
        this.config.baseRetryDelayMs * Math.pow(2, attempt - 1),
        this.config.maxRetryDelayMs
      );
      // 429 Retry-After ヘッダーがあれば優先
      const actualDelay = retryAfterSeconds > 0 ? Math.max(delayMs, retryAfterSeconds * 1000) : delayMs;
      await this.sleep(actualDelay);
    }

    try {
      const result = await this.doCompletion(systemPrompt, userMessage);
      return result;
    } catch (err) {
      lastError = err instanceof Error ? err : new Error(String(err));

      if (lastError instanceof OpenRouterError) {
        lastErrorClass = lastError.openRouterErrorClass;
        retryAfterSeconds = lastError.retryAfterSeconds;
        if (!lastError.retriable) {
          throw lastError;
        }
      } else {
        // 古い形式（互換）: unknown扱いで再試行、401/403だけ明示中断
        const msg = lastError.message;
        if (msg.includes("401") || msg.includes("403")) {
          throw lastError;
        }
      }

      // リトライ可能 → continue（バックオフ後に再試行）
      console.warn(
        `[OpenRouter] Attempt ${attempt + 1}/${this.config.maxRetries + 1} failed ` +
        `[${lastErrorClass}]: ${lastError.message.slice(0, 200)}. Retrying...`
      );
    }
  }

  throw lastError ?? new Error("OpenRouter chat completion failed after retries");
}
```

## 6. Fix 3: フォールバックモデル機構（Phase 2）

### 6.1 設計

| 条件 | プライマリモデル | フォールバックモデル |
|------|-----------------|---------------------|
| ユーザー指定なし（デフォルト） | `openrouter/free` | なし（既にフリー） |
| ユーザーがカスタムモデル指定 | カスタムモデル | `openrouter/free` |

**フロー**:
1. プライマリモデルで12回再試行（3s～600s指数バックオフ）
2. プライマリが全滅 → フォールバックモデルで3回再試行（3s～48s指数バックオフ）
3. フォールバックも全滅 → `null` 返却 → `cacheRetry`

### 6.2 NarrativeWorker への変更

`narrativizeWithRetry()` を2段フェーズに変更:

```typescript
private async narrativizeWithRetry(item: CacheItem): Promise<NarrativeResult | null> {
  const models = this.getRetryModels();

  for (const { model, maxAttempts, label } of models) {
    console.log(`[NarrativeWorker] Phase "${label}": trying model=${model}, maxAttempts=${maxAttempts}`);

    const result = await this.narrativizeWithModel(item, model, maxAttempts, label);
    if (result) return result;

    if (model !== models[models.length - 1].model) {
      console.warn(
        `[NarrativeWorker] Phase "${label}" exhausted for [${item.id}]. Falling back to next model...`
      );
    }
  }

  return null;
}

private getRetryModels(): Array<{ model: string; maxAttempts: number; label: string }> {
  const primary = this.config.openrouterModel ?? "openrouter/free";
  const fallback = "openrouter/free";

  if (primary === fallback) {
    // デフォルト: openrouter/free のみ → 12回
    return [{ model: primary, maxAttempts: MAX_RETRIES, label: "primary" }];
  }

  // カスタムモデル指定: プライマリ12回 + フォールバック3回
  return [
    { model: primary, maxAttempts: MAX_RETRIES, label: "primary" },
    { model: fallback, maxAttempts: FALLBACK_RETRIES, label: "fallback" },
  ];
}

private async narrativizeWithModel(
  item: CacheItem,
  model: string,
  maxAttempts: number,
  label: string,
): Promise<NarrativeResult | null> {
  const systemPrompt = this.resolveSystemPrompt();
  const previous = this.config.narrativePreviousEpisodeRef !== false
    ? this.lastNarrativeByAgent.get(item.agentId)
    : undefined;
  const conversationText = item.rawText;
  const userMessage = this.resolveUserPrompt(previous?.body, conversationText);

  for (let attempt = 0; attempt < maxAttempts; attempt++) {
    try {
      const rawText = await this.client.chatCompletion(
        { systemPrompt, userMessage },
        { modelOverride: model }
      );
      // ... quality gates ...
      return { text, tokens, model };
    } catch (err) {
      const delayMs = Math.min(RETRY_BASE_DELAY_MS * Math.pow(2, attempt), MAX_RETRY_DELAY_MS);
      console.warn(
        `[NarrativeWorker] ${label} attempt ${attempt + 1}/${maxAttempts} failed for [${item.id}]: ` +
        `${err instanceof Error ? err.message : String(err)}. Retrying in ${delayMs}ms...`
      );
      await this.sleep(delayMs);
    }
  }

  return null;
}
```

## 8. Fix 5: 重複生成の実害抑制（Idempotency Guard）

### 8.1 問題

`saveNarrative()` → `batchIngest()` → `cacheAck()` のフローで:
- `batchIngest()` 成功 → `cacheAck()` 失敗 → キューに残る → 再リース → 再生成 → 同一内容のエピソードが多重保存される

### 8.2 解決策: rawText ハッシュベースの重複検出

```typescript
// NarrativeWorker クラス内
private recentSaveHashes = new Map<string, number>(); // rawText hash → savedAt timestamp
private readonly SAVE_HASH_TTL_MS = 30 * 60 * 1000; // 30分

private hashRawText(rawText: string): string {
  // 高速ハッシュ: 最初の500文字 + 長さ で実質一意
  // crypto.createHash は不要 — 重複検出用途で衝突確率は十分低い
  const prefix = rawText.slice(0, 500);
  let hash = 0;
  for (let i = 0; i < prefix.length; i++) {
    hash = ((hash << 5) - hash + prefix.charCodeAt(i)) | 0;
  }
  return `${hash}_${rawText.length}`;
}

private pruneSaveHashes(): void {
  const now = Date.now();
  for (const [key, ts] of this.recentSaveHashes) {
    if (now - ts > this.SAVE_HASH_TTL_MS) {
      this.recentSaveHashes.delete(key);
    }
  }
}

private async saveNarrative(result: NarrativeResult, item: CacheItem): Promise<void> {
  const rawHash = this.hashRawText(item.rawText);
  const now = Date.now();

  // 重複保存検出
  this.pruneSaveHashes();
  if (this.recentSaveHashes.has(rawHash)) {
    const savedAt = this.recentSaveHashes.get(rawHash)!;
    const ageMin = ((now - savedAt) / 60000).toFixed(1);
    console.warn(
      `[NarrativeWorker] Duplicate save detected for [${item.id}] (rawText hash=${rawHash}, ` +
      `previously saved ${ageMin}min ago). Skipping batchIngest, acknowledging only.`
    );
    await this.rpcClient.cacheAck(item.id, "narrative-worker");
    return;
  }

  try {
    const tags = ["narrative", item.source === "live-turn" ? "auto-segmented" : "cold-start-import"];
    await this.rpcClient.batchIngest([{ ... }], item.agentWs, item.agentId);

    // 重複防止ハッシュを記録
    this.recentSaveHashes.set(rawHash, now);
    this.lastNarrativeByAgent.set(item.agentId, { episodeId: `narrative-${now}`, body: result.text });
  } catch (err) {
    console.error("[NarrativeWorker] Failed to save narrative episode:", err);
    throw err;
  }
}
```

**設計判断**:
- `crypto.createHash("sha256")` ではなく FNV-1風の簡易ハッシュを使用。重複検出用途で衝突確率は十分低い（同一rawTextの最初500文字+長さが衝突する確率 < 10^-9）
- TTL 30分: 30分以内の同じrawTextの重複保存を防止。30分超過後は別エピソードとして扱う（内容が変わっている可能性）
- `pruneSaveHashes()` は `saveNarrative()` の先頭で毎回実行。Mapサイズは実質的に小さい（1時間あたり数件）

## 9. Fix 6: サーキットブレーカー（Phase 2）

> **適用順序方針**: Circuit Breaker は初回リリース範囲から外し、Fix 0-2 が安定した後に第2フェーズで適用する。

### 9.1 設計

```
           ┌──────────────────────────────┐
           │                              │
    失敗数 < 閾値 │       HALF-OPEN              │  成功
   ┌───────────────┤  (1回だけ試行)               ├───────────┐
   │               │                              │           │
   │               └──────────────────────────────┘           │
   │                       │                                   │
   │                  失敗  │ 成功                              │
   │                       │                                   │
   ┌───────┐         ┌─────┴─────┐                      ┌────▼────┐
   │ CLOSED │◄────────┤   OPEN    │  閾値到達            │ CLOSED  │
   │(正常)  │ タイムアウト│  (遮断)   │◄─────── 失敗数 ────┤ (正常)  │
   └───────┘         └───────────┘                      └─────────┘
```

### 9.2 実装

```typescript
// openrouter-client.ts に CircuitBreaker を追加

export class CircuitBreaker {
  private failures = 0;
  private lastFailureTime = 0;
  private state: "closed" | "open" | "half-open" = "closed";

  constructor(
    private readonly threshold: number = 5,     // 連続失敗閾値
    private readonly resetTimeoutMs: number = 300_000, // 5分でリセット(5min)
  ) {}

  get currentState(): string { return this.state; }

  recordSuccess(): void {
    this.failures = 0;
    this.state = "closed";
  }

  recordFailure(): void {
    this.failures++;
    this.lastFailureTime = Date.now();
    if (this.failures >= this.threshold) {
      this.state = "open";
    }
  }

  canAttempt(): boolean {
    if (this.state === "closed") return true;
    if (this.state === "open") {
      // タイムアウト経過 → half-open に移行して1回だけ試行許可
      if (Date.now() - this.lastFailureTime >= this.resetTimeoutMs) {
        this.state = "half-open";
        return true;
      }
      return false;
    }
    // half-open: 1回だけ試行許可
    return true;
  }
}
```

### 9.3 統合

`OpenRouterClient` にサーキットブレーカーを組み込む:

```typescript
export class OpenRouterClient {
  private config: OpenRouterConfig;
  private circuitBreaker = new CircuitBreaker();

  async chatCompletion(
    params: { systemPrompt: string; userMessage: string },
    opts?: { modelOverride?: string },
  ): Promise<string> {
    if (!this.circuitBreaker.canAttempt()) {
      throw new OpenRouterError({
        message:
          `OpenRouter circuit breaker OPEN for model=${this.config.model}. ` +
          `Cooling down before retry.`,
        errorClass: "circuit_open",
        retriable: true, // サーキットブレーカー遮断は一時的
      });
    }

    // ... existing retry loop ...

    // 成功時:
    this.circuitBreaker.recordSuccess();
    return result;

    // 失敗時 (catch 内):
    this.circuitBreaker.recordFailure();
  }
}
```

**サーキットブレーカー遮断時のフォールバック**:
- プライマリモデルが遮断 → フォールバックモデル（openrouter/free）へ自動切り替え
- フォールバックも遮断 → NarrativeWorker の `narrativizeWithRetry()` が `null` 返却 → `cacheRetry` で再キュー

### 9.4 サーキットブレーカーパラメータ

| パラメータ | 値 | 理由 |
|-----------|---|------|
| 閾値（連続失敗数） | 5 | 連続5回失敗で遮断。freeティアの一時障害には猶予あり |
| リセットタイムアウト | 5分 | プロバイダ障害の復旧待ちとして5分は適切 |
| モデル別管理 | プライマリ/フォールバック別々 | 各モデル独立してサーキット状態を持つ |

### 17.4 実装中に判明した設計論点と最終調整

1. **Circuit Breaker共有問題（修正済み）**
   - 初期実装では `OpenRouterClient` 内で単一 breaker を共有していたため、
     custom model の失敗が fallback `openrouter/free` まで巻き込むリスクがあった。
   - 最終実装で `Map<string, CircuitBreaker>` に変更し、**モデル別に独立管理**へ修正。

2. **Circuit Failure加算が過剰になる問題（修正済み）**
   - 内部 retry の各失敗で `recordFailure()` すると breaker が過敏に開く。
   - 最終実装は **chatCompletion 1呼び出しにつき最大1 failure加算**に調整。
     （non-retriable 即時失敗時はその時点で1加算）

3. **modelOverride方式の厳守（反映済み）**
   - `this.config.model` 一時変更案は採用せず、
     `chatCompletion(params, { modelOverride })` のリクエスト単位指定を採用。
