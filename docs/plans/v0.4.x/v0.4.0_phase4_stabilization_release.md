#### Step 0b: `openclaw.plugin.json` 経由でユーザーが調整可能にする

**設計方針:** ユーザーが `openclaw.plugin.json` に `openrouterConfig` オブジェクトで直接指定できる形式にする。OpenClaw の configSchema が自動的にバリデーションをかけてくれるので安全。

**`maxTokens` の設計決定:** OpenRouter の `max_tokens` は **Optional** (公式 doc 確認済み)。

| 方選 | 内容 |
|---|---|
| **省略 (default)** | OpenRouter/モデルに任せる -- 切れるリスクなし、推奨 |
| 指定あり | 値を上限キャップ。冗長出力防止やコスト制御したい場合のみ |

**結論: デフォルトは省略 (`undefined`)** -- ユーザーが必要時にのみ `maxTokens` を設定する。

**完成イメージ (ユーザーの設定例):**
```json
{
  "episodic-claw": {
    "enabled": true,
    "openrouterConfig": {
      "model": "openrouter/free",
      "temperature": 0.5
    }
  }
}
```

> `maxTokens` は省略 = OpenRouter 任せ (推奨)。上限を設けたい場合のみ `"maxTokens": 2048` を追加。

> `apiKey` は `openrouterConfig` には含めない。セキュリティ上 `openrouterApiKey` を既存の flat フィールドとして残す (`secret: true`)。
