## 1. 評価結果サマリー（結論）

### 結論

**採用方針は「KISSハイブリッド」**。

1. 全面的な新規状態機械（大規模構造パーサ）は導入しない
2. 実測で確実に出ている段落境界（`Keep caption...` 以降）を優先利用する

1. 既存バグ（media leak）は修正済みで、今回の主眼は保守性改善 + 残ノイズ源の除去
2. 実セッションで、添付メッセージ構造は高い再現性を示した（45/45一致）
3. ただしチャネル互換性のため、既存フォールバックは全削除しない方が安全
4. 大規模構造パーサ（行ループ + 状態遷移）はステップ数 3→7 に増え、KISSに反する

## 5. ターゲット仕様（実装後の期待挙動）

### 5.1 添付メッセージの抽出仕様

**⚠ 実行順序の制約（最重要）**

現行の `ATTACHMENT_BOILERPLATE` L54:

```typescript
/To send an image back[\s\S]*?Keep caption in the text body\./gi
```

この正規表現が **境界文字列 `Keep caption in the text body.` を含むブロック全体を一括除去** する。

したがって、**tail 優先抽出は必ず ATTACHMENT_BOILERPLATE の regex strip より前に実行**しなければならない。そうしないと境界が消えて tail 抽出が不可能になる。

**`classifyAndStripAttachment()` の実行フロー（実装後）:**

```
1. [EXISTING] MEDIA_ONLY_SENTINEL の早期チェック
   → isDominant=true なら即リターン
2. [NEW] text に "[media attached" と "Keep caption in the text body." の
   両方が含まれるか判定
3. [NEW] 両方ある場合 → "Keep caption in the text body." の lastIndexOf で split
   → tail（境界より後ろ）だけを取り出す
4. [NEW] tail 内ノイズ除去（正規表現）:
   - /^System:\s.*$/gm
   - /^(Conversation info|Sender|Replied message)\s+\(untrusted.*\):/gim
   - fenced json ブロック (```json ... ```)
   - /^media:(image|document|audio|video)\s*$/gim
   - /<media:(image|document|audio|video)>/gi
   - /media:\/\/inbound\/[^\s]+/gi
   - 空白正規化
5. [NEW] tail 抽出結果が empty でなければ、cleanedText = 正規化後のtail として判定へ
6. [EXISTING] 条件に当てはまらない場合 → 従来の ATTACHMENT_BOILERPLATE イテレート
7. [EXISTING] 空白正規化 → 長さ閾値 + INDICATORS 判定
```

**実装上の注意:** ステップ 4 のノイズ除去 regex は `ATTACHMENT_BOILERPLATE` と重複するパターンがあるが、適用対象が「tail のみ」であるため、全体適用の `ATTACHMENT_BOILERPLATE` とは独立して動作する。tail パスを通った場合はステップ 6 をスキップする。

### 5.2 dominant 判定仕様

1. 判定は最終 `cleanedText` のみを見る
2. ルール:
   - CJKあり: `length < 2` なら dominant
   - CJKなし: `length < 5` なら dominant
   - `ATTACHMENT_INDICATORS` カウント > 0 かつ `wordCount <= indicatorCount` なら dominant（セーフティネット）
3. `ATTACHMENT_INDICATORS` はセーフティネットとして維持する（削除しない）

### 5.3 API整合仕様

1. `isAttachmentDominant(text)` は `classifyAndStripAttachment(text).isDominant` へ委譲（ワンライナー）
2. `stripAttachmentNoise(text)` は `classifyAndStripAttachment(text).cleanedText` へ委譲（ワンライナー）
3. これにより判定の真 truth source を `classifyAndStripAttachment` に統一

```typescript
export function classifyAndStripAttachment(text: string): {
  isDominant: boolean;
  cleanedText: string;
} {
  // [EXISTING] Sentinel check first (cheap)
  if (MEDIA_ONLY_SENTINEL.test(text.trim())) {
    return { isDominant: true, cleanedText: "" };
  }

  // [NEW] Phase B: tail-priority extraction
  // MUST run BEFORE ATTACHMENT_BOILERPLATE strip because L54 removes
  // "Keep caption in the text body." as part of the boilerplate block.
  const hasMediaHeader = /\[media attached/i.test(text);
  const hasBoilerplateEnd = /Keep caption in the text body\./i.test(text);

  if (hasMediaHeader && hasBoilerplateEnd) {
    const boundaryIdx = text.lastIndexOf("Keep caption in the text body.");
    if (boundaryIdx !== -1) {
      const afterBoundary = text.slice(boundaryIdx + "Keep caption in the text body.".length);

      let tailCleaned = afterBoundary;
      // Remove System: lines
      tailCleaned = tailCleaned.replace(/^System:\s.*$/gm, "");
      // Remove untrusted metadata headers
      tailCleaned = tailCleaned.replace(
        /^(Conversation info|Sender|Replied message)\s+\(untrusted[^)]*\):.*$/gim, ""
      );
      // Remove fenced json blocks (```json ... ```)
      tailCleaned = tailCleaned.replace(/```json[\s\S]*?```/g, "");
      // Remove bare media:type lines
      tailCleaned = tailCleaned.replace(/^media:(image|document|audio|video)\s*$/gim, "");
      // Remove <media:...> tags
      tailCleaned = tailCleaned.replace(/<media:(image|document|audio|video)>(?:\s*\([^)]*\))?/gi, "");
      // Remove media://inbound/ URIs
      tailCleaned = tailCleaned.replace(/media:\/\/inbound\/[^\s]+/gi, "");
      // Normalize whitespace
      tailCleaned = tailCleaned.replace(/\n{3,}/g, "\n\n").replace(/[ \t]+/g, " ").trim();

      // If tail extraction produced meaningful text, use it as the cleaned result
      if (tailCleaned.length >= 2) {
        const cjkChars = (tailCleaned.match(
          /[\p{Script=Han}\p{Script=Katakana}\p{Script=Hiragana}\p{Script=Hangul}]/gu
        ) || []).length;
        const isDominant = tailCleaned.length < 2 ||
          (cjkChars === 0 && tailCleaned.length < 5);
        return { isDominant, cleanedText: tailCleaned };
      }
      // Tail was empty after cleanup — pure attachment noise
      return { isDominant: true, cleanedText: "" };
    }
  }

  // [EXISTING] Fallback: original regex-based strip for messages without the boilerplate boundary
  let cleaned = text;
  let markerCount = 0;

  for (const pattern of ATTACHMENT_BOILERPLATE) {
    const matches = cleaned.match(pattern);
    if (matches) markerCount += matches.length;
    cleaned = cleaned.replace(pattern, "");
  }

  // ... (rest of existing logic unchanged) ...
```

### 12.1 ギャップ1: tailパスのdominant判定にINDICATORSセーフティネットが欠落

**現状:**

`classifyAndStripAttachment()` の tail パス（retriever.ts L165-171）:

```typescript
const isDominant = tailCleaned.length < 2 ||
  (cjkChars === 0 && tailCleaned.length < 5);
```

一方、フォールバックパス（L194-204）には `ATTACHMENT_INDICATORS` によるセーフティネットがある:

```typescript
const isDominant =
  cleaned.length < 2 ||
  (cjkChars === 0 && cleaned.length < 5) ||
  (indicatorCount > 0 && wordCount <= indicatorCount);  // ← これがtailパスにない
```

**プラン Section 5.2 の仕様:**

> `ATTACHMENT_INDICATORS` カウント > 0 かつ `wordCount <= indicatorCount` なら dominant（セーフティネット）

tailパスにもこの判定が必要と読めるが、実装から漏れている。

**リスク評価:**

実測データ（Section 2）では `Keep caption...` 後に INDICATORS のみが残るケースは 0/45 だった。したがって現状の実害は**極低**。ただし新規チャネルやエッジケースで、caption に見せかけたパス拡張子だけが残るメッセージが来た場合、dominant=false と誤判定する可能性がある。

**修正方針:**

tailパスの dominant 判定に INDICATORS カウントを追加する。修正対象は `classifyAndStripAttachment()` 内の L165-171 のみ。

```typescript
// 修正後の tail パス dominant 判定
if (tailCleaned.length >= 2) {
  const cjkChars = (tailCleaned.match(
    /[\p{Script=Han}\p{Script=Katakana}\p{Script=Hiragana}\p{Script=Hangul}]/gu
  ) || []).length;

  // INDICATORS セーフティネット（フォールバックパスと同一ロジック）
  let indicatorCount = 0;
  for (const pattern of ATTACHMENT_INDICATORS) {
    const matches = tailCleaned.match(pattern);
    if (matches) indicatorCount += matches.length;
  }
  const wordCount = tailCleaned.split(/\s+/).filter(Boolean).length;

  const isDominant = tailCleaned.length < 2 ||
    (cjkChars === 0 && tailCleaned.length < 5) ||
    (indicatorCount > 0 && wordCount <= indicatorCount);

  return { isDominant, cleanedText: tailCleaned };
}
```

**注意:** INDICATORS の正規表現は `gi` フラグ付きのため、`match()` を呼ぶ前に `lastIndex` をリセットする必要はない（関数スコープのローカル変数で評価するため）。ただし `ATTACHMENT_INDICATORS` をループ内で使う場合、`gi` フラグ付き RegExp のステートフル問題（`lastIndex`）に注意すること。現行フォールバックパスでも同じパターンを使っているため、既知の振る舞い。

**テスト追加:**

`test_classify_attachment.ts` に INDICATORS セーフティネットが効くケースを追加:

```typescript
// Case J: tailにINDICATORSのみ残るケース
const msgJ = `[media attached]
Keep caption in the text body.
.jpg .png /home/user/photo`;
```

### 12.2 ギャップ2: test_phase4_5.ts のstandaloneコピーにtailパスが未反映

**現状:**

`test_phase4_5.ts` L2176-2198 の `testIsAttachmentDominant()` は旧来の2段構え（SENTINEL → BOILERPLATE iterate）のまま。tail-priority抽出のロジックが入っていない。

具体的には、本番 `classifyAndStripAttachment()` では以下の経路が存在する:

1. SENTINEL → dominant=true
2. **tail抽出** → `Keep caption...` 以降のクリーンアップ（System:行・untrusted metadata・fenced json 除去）

**リスク評価:**

- System: 行 + caption のメッセージ → 本番はtailパスでSystem:行を除去、standaloneはBOILERPLATE iterate後にSystem:行が残る可能性

- fenced json + caption → 同上

`testIsAttachmentDominant()` / `testStripAttachmentNoise()` に tail パス相当のロジックを追加する。本番との完全一致を保証するため、standalone コピーではなく本番関数の import に切り替えるのが理想だが、`test_phase4_5.ts` の冒頭コメント（L2122-2123）にある通り ESM import 問題を回避するために standalone にしている。

```typescript
  if (nonMarkerText.length < 2) return true;
  if (cjkChars === 0 && nonMarkerText.length < 5) return true;

  let indicatorCount = 0;
  for (const pattern of TEST_ATTACHMENT_INDICATORS) {
    const matches = nonMarkerText.match(pattern);
    if (matches) indicatorCount += matches.length;
  }
  const wordCount = nonMarkerText.split(/\s+/).filter(Boolean).length;
  if (indicatorCount > 0 && wordCount <= indicatorCount) return true;

  return false;
}
```
