# v0.4.7 Runtime Bridge Mode Master Switch Plan

**Parent:** [v0.4.7 Universal Prompt Hook Bridge Plan](./v0.4.7_universal_prompt_hook_bridge_plan.md)  
**Date:** 2026-04-12  
**Target:** episodic-claw `v0.4.7` (plugin-only hot update)

---

## 1. Goal

`toolFirstRecall` 単体ではなく `runtimeBridgeMode` を主スイッチにする。  
1つの設定で CLI/Embedded の挙動を統制し、ロールバックも明示化する。

```json
"runtimeBridgeMode": "auto" // auto | legacy_embedded | cli_universal
```

- `auto`（デフォルト）  
  実行経路に応じて安全側を自動選択する。
- `legacy_embedded`  
  従来Embedded経路を強制する（保守/ロールバック）。
- `cli_universal`  
  `v0.4.6 + v0.4.7` universal bridge 方針を強制する。

---

## 2. Runtime Policy

### 2.1 Mode Matrix

1. `runtimeBridgeMode=cli_universal`
- `before_dispatch` + `message_sent` を ingest 主経路として有効化
- `before_prompt_build` は strict no-op (`{}`) for recall/injection
- `assemble()` は recall注入しない（必要なら anchor のみ）
- `toolFirstRecall.enabled` は runtime で強制 `true` 扱い
- `ep-recall` の query 生成契約は既存 Parse/Rewrite を維持

2. `runtimeBridgeMode=legacy_embedded`
- `before_dispatch` / `message_sent` bridge ingest は無効化
- `before_prompt_build` / `assemble()` の従来経路を使用
- `toolFirstRecall` は従来どおり設定値を尊重（強制しない）

3. `runtimeBridgeMode=auto`（推奨デフォルト）
- CLI実行時は `cli_universal` 相当
- Embedded実行時は `legacy_embedded` 相当
- 判定不能時は安全側で `cli_universal` を優先し、診断ログを出す

### 2.2 Non-negotiables

1. OpenClaw本体は変更しない  
2. Episodic-Clawプラグインのみ変更  
3. `message_sent` の session mapping 不解決は常に safe no-op

---

## 3. Config Schema Changes

`openclaw.plugin.json` + `src/index.ts` の `PluginConfigSchema` に以下を追加:

```json
"runtimeBridgeMode": {
  "type": "string",
  "enum": ["auto", "legacy_embedded", "cli_universal"],
  "default": "auto"
}
```

`src/types.ts` に `RuntimeBridgeMode` 型を追加し、`EpisodicPluginConfig` に組み込む。  
`src/config.ts` に `resolveRuntimeBridgeMode(rawConfig)` を追加する。

---

## 4. Implementation Scope

1. `src/types.ts`
- `RuntimeBridgeMode = "auto" | "legacy_embedded" | "cli_universal"` を追加
- `EpisodicPluginConfig` に `runtimeBridgeMode?: RuntimeBridgeMode`

2. `src/config.ts`
- `resolveRuntimeBridgeMode(rawConfig)` を追加
- デフォルト `auto`

3. `src/index.ts`
- `PluginConfigSchema` に `runtimeBridgeMode` を追加
- register初期化時に `runtimeBridgeMode` を解決
- 実行時分岐:
  - `cli_universal` の場合のみ bridge ingress (`before_dispatch`/`message_sent`) を有効
  - `legacy_embedded` は bridge ingress を登録しない
  - `auto` は provider/実行経路で上記を切替
- `toolFirstRecall` は `cli_universal`（または `auto` でCLI判定時）に runtime 強制ON
- `before_prompt_build` / `assemble()` の no-op条件を `runtimeBridgeMode` 前提で分岐

4. `openclaw.plugin.json`
- `configSchema.properties.runtimeBridgeMode` を追記
- `toolFirstRecall` の説明文を「`runtimeBridgeMode`優先」の契約に更新

5. Tests
- `test_v047_bridge.ts`: `runtimeBridgeMode` の3モード分岐を追加検証
- `test_tool_first.ts`: `cli_universal` と `auto+CLI` で tool-first 強制ONを追加検証

---

## 5. Acceptance Criteria

1. `runtimeBridgeMode=cli_universal` で CLI/Embedded 切替時も重複ingestなし  
2. `runtimeBridgeMode=cli_universal` で `before_prompt_build` no-op + bridge主経路が成立  
3. `runtimeBridgeMode=legacy_embedded` で従来Embedded挙動が維持  
4. `runtimeBridgeMode=auto` で CLI と Embedded が意図どおり自動切替  
4. Parse/Rewrite query 契約は維持（raw最新1件直投げにしない）  
5. `message_sent` mapping不一致は no-op安全落ち

---

## 6. Risk Notes

1. `auto` 判定が不正確だと誤モードで動く  
2. 判定不能時の挙動は `cli_universal` 優先 + warning ログ必須  
3. 既存ユーザー向けに release notes で migration を明記する
