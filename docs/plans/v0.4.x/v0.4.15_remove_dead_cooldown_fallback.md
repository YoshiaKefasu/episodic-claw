## 1. 問題定義

### 二重フォールバックの構造

`recallReInjectionCooldownTurns` には2段階のデフォルト値が存在する:

```
[ユーザー設定なし]
  → loadConfig() L94: rawConfig?.recallReInjectionCooldownTurns ?? 24  → cfg.recallReInjectionCooldownTurns = 24
    → index.ts L552: cfg.recallReInjectionCooldownTurns ?? 10  → 到達不能（24 は truthy なので `?? 10` は評価されない）
```

| レベル | 場所 | デフォルト | 役割 |
|--------|------|-----------|------|
| 第1段 | `config.ts` L94 | `?? 24` | `loadConfig()` による公式デフォルト |
| 第2段 | `index.ts` L552 | `?? 10` | デッドフォールバック（到達不能） |

### なぜ `?? 10` は到達不能か

`loadConfig()` は `recallReInjectionCooldownTurns` に必ず `24` 以上の値を返す:

```typescript
// config.ts L94
recallReInjectionCooldownTurns: Math.max(0, rawConfig?.recallReInjectionCooldownTurns ?? 24),
```

- ユーザーが設定しない → `?? 24` → `Math.max(0, 24)` → `24`
- ユーザーが `0` を設定 → `Math.max(0, 0)` → `0`
- ユーザーが正の値を設定 → その値（`Math.max` で非負保证）

`0` は falsy なので `?? 10` が発動する可能性があるが、これはユーザーの意図的設定（「クールダウン無効化」）を上書きしてしまう副作用がある（後述）。

### `?? 10` が仮に発動した場合の問題

ユーザーが `recallReInjectionCooldownTurns: 0` を設定した場合:

```
loadConfig() → Math.max(0, 0) → 0
  → cfg.recallReInjectionCooldownTurns = 0
    → index.ts L552: 0 ?? 10 → 10  ← ユーザーの「無効化」意図を無視！
```

これはv0.4.13で追加された「`0` でガード無効化」機能と直接矛盾する（`openclaw.plugin.json` L135 の説明: "Set to 0 to disable"）。

## 2. 修正内容

**変更前**:
