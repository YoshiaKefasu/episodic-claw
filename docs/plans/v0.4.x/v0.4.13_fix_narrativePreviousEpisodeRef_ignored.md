## 1. 問題定義

### 現状

`narrativePreviousEpisodeRef` は3カ所で定義・パースされているが、**実際に消費される場所がゼロ**:

| 場所 | ファイル | 行 | 役割 |
|------|----------|-----|------|
| 型定義 | `types.ts` | L121 | `narrativePreviousEpisodeRef?: boolean` |
| パース | `config.ts` | L105 | `narrativePreviousEpisodeRef: rawConfig?.narrativePreviousEpisodeRef ?? true` |
| スキーマ | `openclaw.plugin.json` | L247-250 | `Type.Optional(Type.Boolean({...}))` |

しかし `narrative-worker.ts` では:

```typescript
// L257 — previous を無条件で取得
const previous = this.lastNarrativeByAgent.get(item.agentId);
// L259 — previous?.body を無条件でプロンプトに注入
const userMessage = this.resolveUserPrompt(previous?.body, conversationText);
```

`resolveUserPrompt` も無条件:

```typescript
// L339-347
private resolveUserPrompt(previousEpisode: string | undefined, conversationText: string): string {
    const custom = this.config.narrativeUserPromptTemplate;
    if (custom && custom.trim().length > 0) {
      return custom
        .replace("{previousEpisode}", previousEpisode || "")
        .replace("{conversationText}", conversationText);
    }
    return DEFAULT_USER_PROMPT_TEMPLATE(previousEpisode, conversationText);
  }
```

### 影響

- ユーザーが `narrativePreviousEpisodeRef: false` を設定しても、前回エピソードが常にLLMに注入される
- 無駄なトークン消費（前回エピソードは200-800文字/呼び出し）
- データ破損ではないが、設定が実質 no-op — **サイレントな設定無視**

## 2. 修正内容

### 2.1 変更対象

`src/narrative-worker.ts` — `narrativizeWithRetry()` の L257

### 2.2 変更後

```typescript
// L257 — respect narrativePreviousEpisodeRef config
const previous = this.config.narrativePreviousEpisodeRef
  ? this.lastNarrativeByAgent.get(item.agentId)
  : undefined;
```

1行の変更。`resolveUserPrompt` 側は変更不要（`previousEpisode: undefined` を受けると `previousEpisode ?` が falsy になるので "Previous episode:" ブロックが挿入されない）。

### 🔍 追加分析: `initContinuity()` への影響

`initContinuity()`（L173-197）でも `this.lastNarrativeByAgent` を更新している。ここは `narrativePreviousEpisodeRef` の影響を受けるべきか？

### 🎯 実装差分（プラン vs 実際）

プランの提案:
```typescript
const previous = this.config.narrativePreviousEpisodeRef
  ? this.lastNarrativeByAgent.get(item.agentId)
  : undefined;
```

実際の実装:
```typescript
const previous = this.config.narrativePreviousEpisodeRef !== false
  ? this.lastNarrativeByAgent.get(item.agentId)
  : undefined;
```

**判定: ✅ 実装の方が優れている**

理由:
- プランの `truthy check` は `undefined`（未設定）を `false` 扱いにするリスクがあった。`config.ts` で `?? true` しているので通常は `undefined` にならないが、コードで直接configオブジェクトを構築した場合に差が出る
- `!== false` は「明示的に `false` と設定されたときだけ無効化」するセマンティクス。デフォルト有効の設定として正しい振る舞い
- コメント（L257-258）も丁寧。「Use !== false so that undefined (unset) defaults to including previous episode」と理由を明記

### ✅ 総合判定

> **v0.4.13 実装は完了。プラン以上の品質。** `!== false` の採用は設定のデフォルト挙動を正しく表現している。`resolveUserPrompt()` は変更なし（Option A準拠）。追加テストもYAGNIで不要。
