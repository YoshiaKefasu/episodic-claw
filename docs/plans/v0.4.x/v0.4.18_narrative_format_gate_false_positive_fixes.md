## 1. 問題定義

### 1.1 v0.4.17 コードレビューで発見された5件の指摘

| # | 深刻度 | 指摘 | 影響 |
|---|--------|------|------|
| 1 | 🔴 **Bug** | `cotPrefixPat` の `first` が過剰マッチ | 正規の物語「First, he walked to the store.」がCoT判定され削除される |
| 2 | 🟡 **Bug** | Gate 5 `narrativeStartPat` に数字が含まれていない | 「2026年の冬、」「3月15日、」等の日付始まりの物語が弾かれる |
| 3 | 🟡 **FP** | `emojiPat` の `≧∇≦` が個別文字としてマッチ | 技術文脈の「delta ≧ 0」が絵文字判定されRetryされる |
| 4 | 🟠 **整合性** | CHANGELOG の日付が逆転 | `[0.4.17] - 2026-04-17` が `[0.4.16] - 2026-04-19` より前の日付 |
| 5 | 🟢 **Cosmetic** | `normalizeOpenRouterReasoning()` Rule d のJSDocが旧記述 | コメントは「include exclude only when true」だが実装は `raw.exclude !== false` |

### 1.2 Bug #1 の詳細

`sanitizeNarrativeOutput()` の Step 2.5:

```typescript
const cotPrefixPat = /^(?:(?:Okay[,.]?\s*)?(?:let me|I need|I should|I'll|first)[^.]*\.\s*\n?)+/im;
```

`first` が単体でマッチする。`m` フラグにより行頭でマッチするため、以下の正規物語文が削除される：

- ✗ `First, he walked to the store.`
- ✗ `First the rain came, then the wind.`
- ✗ `First he tried the obvious fix.`

**修正**: `first` → `first,?\s+I` に変更し、一人称代名詞 `I` を必須にする。CoT の `First, I need to...` はマッチするが、三人称物語の `First, he...` はマッチしない。

### 1.3 Bug #2 の詳細

`checkNarrativeFormat()` Gate 5:

```typescript
const narrativeStartPat = /^[\p{Script=Hiragana}\p{Script=Katakana}\p{Script=Han}A-Z\u00C0-\u017F"「『]/u;
```

数字 `0-9` が含まれていない。以下の正規物語が弾かれる：

- ✗ `2026年の冬、彼は新しいプロジェクトに着手した。`
- ✗ `3月15日、ヨシアは電脳相棒の設計図を描き始めた。`
- ✗ `5年前のあの日、すべてが変わった。`

**修正**: `0-9` を `narrativeStartPat` の文字クラスに追加。

### 1.4 False Positive #3 の詳細

```typescript
const emojiPat = /[\p{Emoji_Presentation}\u{1F600}-\u{1F64F}\u{1F300}-\u{1F5FF}\u{2600}-\u{26FF}\u{2700}-\u{27BF}≧∇≦]/u;
```

`≧∇≦` は文字クラス `[...]` 内にあるため、`≧`・`∇`・`≦` が**個別に**マッチする。技術文脈の `delta ≧ 0` が絵文字判定される。

**修正**: `≧∇≦` を文字クラスから削除し、alternation `(?:≧∇≦)` として全文一致に変更する。

### 1.5 整合性 #4 の詳細

CHANGELOG.md の日付：

```
## [0.4.17] - 2026-04-17    ← 正しい（今日）
## [0.4.16] - 2026-04-19    ← 未来日付（誤り）
```

v0.4.16 の日付 `2026-04-19` が未来日付になっている。実際のリリース日（2026-04-17 以前）に修正する必要がある。

### 1.6 Cosmetic #5 の詳細

`normalizeOpenRouterReasoning()` の Rule d コメント：

```typescript
// Rule d: include exclude only when true
```

現在の実装は `raw.exclude !== false` で `exclude: true` をデフォルト化しているが、コメントは旧仕様（`raw.exclude === true` の時のみ）のままである。

**修正**: コメントを `Rule d [v0.4.17]: Default exclude=true for narrative path — only omit when user explicitly sets exclude: false` に更新。

### 2.2 修正詳細

#### Fix 1: `cotPrefixPat` の `first` 過剰マッチ修正

**現行** (narrative-worker.ts L79-83):

```typescript
const cotPrefixPat = /^(?:(?:Okay[,.]?\s*)?(?:let me|I need|I should|I'll|first)[^.]*\.\s*\n?)+/im;
```

**変更後**:

```typescript
const cotPrefixPat = /^(?:(?:Okay[,.]?\s*)?(?:let me|I need|I should|I'll|first,?\s+I)[^.]*\.\s*\n?)+/im;
```

**変更点**: `first` → `first,?\s+I`。CoT の `First, I need to...` はマッチするが、三人称の `First, he...` はマッチしない。

**影響**: `checkNarrativeFormat()` Gate 2 の CoT パターンにも同様の `first` があるが、Gate 2 は `\bFirst,?\s+I\s+(need|should|will|must)\b` であり、既に `I` が必須のため問題なし。修正不要。

#### Fix 2: `narrativeStartPat` に数字を追加

**現行** (narrative-worker.ts L190-191):

```typescript
const narrativeStartPat = /^[\p{Script=Hiragana}\p{Script=Katakana}\p{Script=Han}A-Z\u00C0-\u017F"「『]/u;
```

**変更後**:

```typescript
const narrativeStartPat = /^[\p{Script=Hiragana}\p{Script=Katakana}\p{Script=Han}0-9A-Z\u00C0-\u017F"「『]/u;
```

**変更点**: `0-9` を追加。日付・年号始まりの物語が正当に PASS する。

**影響**: 数字始まりのアシスタント出力（稀）も PASS するようになるが、数字で始まるアシスタント出力は実質存在しない（アシスタントは「1.」「2026年...」ではなく「ありがとうございます」「Okay」で始まる）。実害なし。

#### Fix 3: `emojiPat` の kaomoji マッチング修正

**現行** (narrative-worker.ts L179-183):

```typescript
const emojiPat = /[\p{Emoji_Presentation}\u{1F600}-\u{1F64F}\u{1F300}-\u{1F5FF}\u{2600}-\u{26FF}\u{2700}-\u{27BF}≧∇≦]/u;
```

**変更後**:

```typescript
const emojiPat = /[\p{Emoji_Presentation}\u{1F600}-\u{1F64F}\u{1F300}-\u{1F5FF}\u{2600}-\u{26FF}\u{2700}-\u{27BF}]|≧∇≦/u;
```

**変更点**: `≧∇≦` を文字クラス `[...]` から alternation `|` に移動。3文字の全文一致のみマッチするようになる。`≧` 単体はマッチしない。

**影響**: 技術文脈の `delta ≧ 0` が絵文字判定されなくなる。`≧∇≦` のみがkaomojiとして検出される。

#### Fix 4: CHANGELOG 日付修正

v0.4.16 の日付 `2026-04-19` が未来日付になっている。実際のリリース日に合わせて修正する。GitHub Release の作成日に基づく。

#### Fix 5: JSDoc コメント更新

**現行** (config.ts Rule d コメント):

```typescript
// Rule d: include exclude only when true
```

**変更後**:

```typescript
// Rule d [v0.4.17]: Default exclude=true for narrative path — prevents CoT token leakage into output
// Only disable if user explicitly sets exclude: false
```

## 6. リスク評価

- **リスク**: **極低**。5件ともピンポイントの正規表現修正・コメント更新・日付修正。ロジックの構造変更なし。

- **ロールバック**: 各修正は独立してリバート可能。
