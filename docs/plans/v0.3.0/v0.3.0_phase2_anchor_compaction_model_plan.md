# v0.3.0 Phase 2 プラン - Anchor compaction モデルを作る

- 状態: 収束済み（現行 baseline の整理）

## 現行実装（v0.3.0 baseline）
- 現行 compactor は、session を heuristic な `Compaction Anchor + Compaction Summary + freshTail` に rewrite する。
- `freshTailCount` が canonical name、`recentKeep` は compatibility alias のまま残る。
- dedicated lossless raw archival、Agent / internal LLM-authored Anchor、`maxAnchorTokens` / `contextThreshold` runtime wiring は **このフェーズの current behavior ではなく target / deferred**。`anchorPrompt` / `compactionPrompt` のテンプレート配線だけは現行コードに入っている。

## target / deferred
- oldest chunk を dedicated lossless raw archive として保持すること
- Anchor を Agent / internal OpenClaw LLM に書かせること
- `maxAnchorTokens` / `contextThreshold` を runtime wiring すること

## 目的
- compaction 本体を「古い履歴の単純退避」から「Anchor を残しながら窓を軽くする処理」へ変える。
- active context を `Anchor + compaction summary + freshTail` に保つ。

## このフェーズで決めること
- oldest chunk の切り出し規則
- `freshTailCount = 96` の保護方法
- heuristic anchor / summary baseline をどこまで current behavior として固定するか
- target / deferred に逃がす項目をどこで線引きするか

## 現行コードとの関係
- [compactor.ts](/D:/GitHub/OpenClaw%20Related%20Repos/episodic-claw/src/compactor.ts) は、現在は古い会話を退避し、heuristic な `Compaction Anchor` と `Compaction Summary`、それに `freshTailCount` 件の raw メッセージを残している。実装互換の legacy alias として `recentKeep` は残るが、このフェーズでは `freshTailCount` を正名として扱う。
- [segmenter.ts](/D:/GitHub/OpenClaw%20Related%20Repos/episodic-claw/src/segmenter.ts) の `summarizeBuffer()` は LLM 要約ではなく、raw text 連結が中心。
- このフェーズでは、現行の heuristic anchor / heuristic summary を、Phase 2 の Anchor compaction モデルへ置き換える。

## 実装タスク
- protected fresh tail を超えた oldest chunk を切り出す。
- 同じ chunk を材料に heuristic anchor を生成する。
- Anchor とは別に heuristic compaction summary を生成する。
- session rewrite 時には raw chunk を落とし、`Anchor + summary + freshTail` を残す。

## 設計方針
- 現行 v0.3.0 baseline の Anchor は heuristic text 生成だが、橋渡し文面のテンプレートとして `anchorPrompt` / `compactionPrompt` は設定配線済み。
- `anchorPrompt` に従って Agent が自由に作文する設計は、引き続き target / deferred として扱う。
- `freshTailCount` を正名にし、`recentKeep` は移行期間の compatibility alias として扱う。既定値は `96` を基準にそろえ、`Compactor` の直接生成時も同じ既定値を使う。
- compaction summary は Anchor の代用にしない。
- D1 consolidation は後段処理として残し、即時 compaction の代役にはしない。

## 受け入れ条件
- compaction 後に active context が `Anchor + summary + freshTail` になる。
- Anchor と summary が別責務として扱われる。
- `freshTailCount` を正名として扱い、`recentKeep` は compatibility alias のまま既存設定を壊さない。

## 非対象
- `prependSystemContext` 注入寿命
- 注入ログ
- `ep-save` の仕様
- recall と lexical の整合
- `contextThreshold` の runtime wiring
- `/compact` command surface の ownership（Phase 4 で扱う）

## 履歴監査ログ
- 以下は Phase 2 を収束させるまでの research / audit history。
- current behavior と target / deferred の正本は、上の `現行実装（v0.3.0 baseline）` と `target / deferred` を優先する。

### Findings
- `lossless-claw` は public plugin hook としての `after_turn` を前提にしているのではなく、`src/engine.ts:2157-2255` の `afterTurn(...)` という **engine 内部の post-turn lifecycle** で、`ingestBatch()` の後に `evaluateLeafTrigger()` と `compact()` を順に呼んでいます。したがって、Phase 2 では **lifecycle の参考実装** と **OpenClaw public hook 契約** を分けて書く必要があります。根拠: [lossless-claw/src/engine.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/engine.ts:2193), [lossless-claw/src/engine.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/engine.ts:2226), [lossless-claw/src/engine.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/engine.ts:2244)
- `contextThreshold` の評価式は `currentTokens > floor(contextThreshold * tokenBudget)` で、そのまま `CompactionDecision` の `threshold` に入ります。これは Phase 2 のしきい値定義としてそのまま使えます。根拠: [lossless-claw/src/compaction.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/compaction.ts:347), [lossless-claw/src/compaction.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/compaction.ts:360), [lossless-claw/src/compaction.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/compaction.ts:362)
- protected fresh tail を超えた oldest raw chunk を切り出す流れは、`resolveFreshTailOrdinal()` で境界を決め、その外側にある最古の連続 message 群だけを `selectOldestLeafChunk()` で拾う形です。Phase 2 の「oldest chunk の切り出し規則」はこれに対応します。根拠: [lossless-claw/src/compaction.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/compaction.ts:801), [lossless-claw/src/compaction.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/compaction.ts:858), [lossless-claw/src/compaction.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/compaction.ts:864)
- `evaluateLeafTrigger()` は full threshold 判定の代わりではなく、**fresh tail の外側に残った raw message token** が `leafChunkTokens` を超えたら「軽い leaf compaction を1回だけ走らせる」ための soft trigger です。Phase 2 の言葉では「overflow 本体」ではなく「incremental leaf compaction の前段判定」です。根拠: [lossless-claw/src/compaction.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/compaction.ts:379), [lossless-claw/src/compaction.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/compaction.ts:386), [lossless-claw/src/engine.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/engine.ts:2226)
- `compact()` は実質 `compactFullSweep()` の入口で、leaf pass を oldest raw chunk から繰り返し、続いて condensed pass を浅い depth から進めます。一方 `compactUntilUnder()` は target token 以下になるまで `compact()` を round 単位で繰り返す budget-targeted recovery です。Phase 2 の言葉では `compact()` が **1 回の compaction sweep**、`compactUntilUnder()` が **しきい値超過からの収束ループ** に当たります。根拠: [lossless-claw/src/compaction.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/compaction.ts:402), [lossless-claw/src/compaction.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/compaction.ts:412), [lossless-claw/src/compaction.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/compaction.ts:696), [lossless-claw/src/compaction.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/compaction.ts:728)
- `lossless-claw` は raw history を消さず、`context_items` 上で source range を summary item に置き換えることで active context だけを軽くしています。Phase 2 の `Anchor + compaction summary + freshTail` は、この「context list を compact に保ちつつ ordering を残す」発想に対応しますが、`lossless-claw` 自体は Anchor を持たず summary 中心です。根拠: [lossless-claw/docs/architecture.md](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/docs/architecture.md:47), [lossless-claw/docs/architecture.md](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/docs/architecture.md:49), [lossless-claw/docs/architecture.md](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/docs/architecture.md:124), [lossless-claw/docs/architecture.md](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/docs/architecture.md:126)
- `contextThreshold` と `freshTailCount` は設定値として独立していて、デフォルトは `0.75` と `64` です。Phase 2 の `0.85` / `96` は別のポリシー値として置けますが、**threshold は budget 比率、fresh tail は raw message 数** という意味づけはこのまま借りるのが自然です。根拠: [lossless-claw/src/db/config.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/db/config.ts:13), [lossless-claw/src/db/config.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/db/config.ts:149), [lossless-claw/src/db/config.ts](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/src/db/config.ts:152), [lossless-claw/docs/configuration.md](D:/GitHub/OpenClaw%20Related%20Repos/lossless-claw/docs/configuration.md:39)
- `openclaw-source` 側には public plugin hook としての `after_turn` は存在しません。したがって、Phase 2 文書では `lossless-claw afterTurn` を **内部 lifecycle の参考** と明記し、public hook 契約は Phase 1 で固定した `before_prompt_build` / `before_compaction` / `after_compaction` 側へ分離して扱うのが必要です。根拠: [openclaw-source/src/plugins/types.ts](D:/GitHub/OpenClaw%20Related%20Repos/openclaw-source/src/plugins/types.ts:1998), [openclaw-source/src/plugins/types.ts](D:/GitHub/OpenClaw%20Related%20Repos/openclaw-source/src/plugins/types.ts:2070)

### 修正や実装方針
- Phase 2 では `lossless-claw afterTurn` を「compaction を post-turn で起動する内部 lifecycle の参考」とだけ扱い、OpenClaw public hook として移植する前提では書かない。
- `contextThreshold` の説明は **`currentTokens > floor(contextThreshold * tokenBudget)`** と明文化し、full threshold 判定と leaf soft trigger を分けて書く。
- `evaluateLeafTrigger()` は「fresh tail 外の raw token が `leafChunkTokens` を超えたときに oldest raw chunk へ 1 回 leaf compaction を打つ前段判定」として、full compaction とは別物だと整理する。
- `compact()` は Phase 2 の言葉では「1 sweep」、`compactUntilUnder()` は「target token 以下まで sweep を繰り返す overflow recovery」と翻訳しておく。
- `lossless-claw` の summary 置換モデルは、Phase 2 では `Anchor + compaction summary + freshTail` の設計へ読み替えるが、Anchor 自体は新規導入物として明確に分ける。

### Resolution
- `lossless-claw` の compaction lifecycle から、threshold 判定・fresh tail 保護・oldest raw chunk selection・budget-targeted convergence の事実を抽出した。
- `afterTurn` は internal lifecycle、OpenClaw public hook は別契約、という境界を Phase 2 に追記できる材料が揃った。
- Phase 2 の `raw history は lossless に残しつつ active context を軽くする` という狙いは、`lossless-claw` の `context_items` 置換モデルと整合する。
- ただし `Anchor` は `lossless-claw` に無いので、Phase 2 では summary 中心モデルをそのまま写すのではなく、Anchor を追加した差分設計として扱う。

### Recommendation
Phase 2 は **まだ開いておくべき**です。`lossless-claw` から compaction モデルの骨格は十分取れましたが、`episodic-claw` 側で Anchor と compaction summary をどの保存単位で分けるかは、この文書でまだ確定していません。
