# v0.3.0 Phase 1 プラン - Hook 契約を固定する

- 状態: 収束済み

## 目的
- Anchor compaction を載せる前に、OpenClaw 側の実在 hook 契約を固定する。
- prompt injection は `before_prompt_build` 主体、`before_agent_start` は `systemPrompt` の legacy fallback として扱う。
- compaction は `before_compaction` / `after_compaction` の実在フックで扱い、`after_turn` 前提は置かない。

## このフェーズで決めること
- OpenClaw source に `after_turn` の plugin hook が存在しないことを前提に、どの hook を採用するか。
- `before_prompt_build` と `before_agent_start` の責務分離をどう固定するか。
- compaction 側は `before_compaction` / `after_compaction` のどちらで何を扱うか。
- `before_ingest` は現行 `PluginHookName` に存在しないため、Phase 1 では枝として残さない。

## 現行コードとの関係
- [openclaw-source/src/plugins/types.ts](/D:/GitHub/OpenClaw%20Related%20Repos/openclaw-source/src/plugins/types.ts) の `PluginHookName` / `PROMPT_INJECTION_HOOK_NAMES` には `after_turn` が無い。
- [openclaw-source/src/plugins/hooks.ts](/D:/GitHub/OpenClaw%20Related%20Repos/openclaw-source/src/plugins/hooks.ts) には `runBeforePromptBuild`, `runBeforeAgentStart`, `runBeforeCompaction`, `runAfterCompaction` がある。
- [openclaw-source/src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts](/D:/GitHub/OpenClaw%20Related%20Repos/openclaw-source/src/agents/pi-embedded-runner/run/attempt.prompt-helpers.ts) で `prependContext` / `prependSystemContext` は `before_prompt_build` と `before_agent_start` の両方から合流する。
- [openclaw-source/src/agents/pi-embedded-runner/run/attempt.ts](/D:/GitHub/OpenClaw%20Related%20Repos/openclaw-source/src/agents/pi-embedded-runner/run/attempt.ts) では `systemPrompt` override を先に適用したうえで `composeSystemPromptWithHookContext()` に渡す。
- [episodic-claw/src/index.ts](/D:/GitHub/OpenClaw%20Related%20Repos/episodic-claw/src/index.ts) 側は `before_prompt_build` を前提に注入を組み、compaction は別レーンに分離する必要がある。

## 調査タスク
- OpenClaw docs と host 実装で、`before_prompt_build` と `before_agent_start` の責務境界を再確認する。
- compaction hook は `contextEngine.info.ownsCompaction === true` のときだけ runner に乗る前提を確認し、`before_compaction` / `after_compaction` のどちらに何を置くかを確定する。
- `prependSystemContext` は prompt 注入の最終合流点として扱い、compaction 判定とは混ぜない。
- `before_ingest` は現行 API では存在しないため、このフェーズでは追わない。

## 設計方針
- hook は増やしすぎない。
- prompt 注入は `before_prompt_build` に寄せる。
- `before_agent_start` は legacy fallback として、ただし `prependContext` / `prependSystemContext` の合流対象として扱う。
- compaction 判定と prompt 注入は別レーンに分ける。
- compaction 側の state 更新は `contextEngine.info.ownsCompaction === true` の経路で完結させる。
- 実際の注入は後続フェーズの `prependSystemContext` 側で扱い、`systemPrompt` override の後に積む。

## 受け入れ条件
- `after_turn` は public plugin hook としては採用しない、という結論が明記されている。
- `before_prompt_build` 主体、`before_agent_start` は legacy fallback だが合流はする、という契約が確定している。
- compaction は `contextEngine.info.ownsCompaction === true` のときだけ `before_compaction` / `after_compaction` に閉じている。
- `prependSystemContext` は `systemPrompt` override の後に合流する最終注入点として整理されている。
- このフェーズの結論だけで、次の Anchor compaction 実装に進める。

## 非対象
- Anchor の文面生成
- compaction summary の中身
- `ep-save` の入口整理
- semantic / lexical の索引同期
- `/compact` command ownership の最終判断（Phase 4 で扱う）
