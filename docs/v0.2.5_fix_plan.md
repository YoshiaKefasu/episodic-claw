# v0.2.5 修正プラン — Episodic-Claw 現行契約

## 目的
- 自動注入（remember / recall）が安定して system prompt に入ることを保証する。
- 失敗時に system prompt を汚染しない。
- 直近文脈の変化に追従できる recall キャッシュにする。
- 各 agent は自分の workspace だけを使い、shared / cross-agent recall は使わない。

## 現行 runtime 契約
1. `sharedEpisodesDir` と `allowCrossAgentRecall` は schema には残してあるが、runtime では無効である。
2. 各 agent は `resolveAgentWorkspaces()` で自分の `agentWs` を解決し、その workspace だけを使う。
3. `assemble()` / `ep-recall` / `ep-expand` / `ep-save` / watcher はすべて agent workspace 基準で動く。
4. `retriever` は agent workspace だけを検索し、legacy / shared workspace を併読しない。
5. `recallFeedback` は agent workspace にだけ帰属させる。

## 実装済みの挙動
### 1) 共有 / 横断 recall の無効化
- `sharedEpisodesDir` は runtime で無視する。
- `allowCrossAgentRecall` は runtime で無視する。
- `resolveAgentWorkspaces()` は `agentId` と `agentWs` を解決するだけで、shared / legacy workspace は返さない。
- watcher は agent workspace のみを起動対象にする。

### 2) recall / inject の安定化
- recall キャッシュキーは `agentId + agentWs + maxEpisodicTokens + k + sha1(fullQuery)`。
- `assemble()` は `segmenter.processTurn()` を fire-and-forget で投げる。
- `maxEpisodicTokens <= 0` の場合は注入しない。
- recall 失敗時は空文字を返し、system prompt にエラー文を混ぜない。

### 5.1) freshness contract
- `assemble()` は `segmenter.processTurn()` を fire-and-forget で投げるため、ingest boundary 直後の `same query immediately after ingest boundary` は **eventual freshness** を前提にする。
- その窓は watcher / debouncer / background index の完了で閉じる。即時反映を保証するのではなく、cache invalidation と再検索で追従する。

### 3) explicit tools の契約
- `ep-recall` は agent workspace だけを検索する。
- `ep-expand` は agent workspace だけを展開する。
- `ep-save` は workspace 単位で recall cache を無効化する。
- `retriever` の recall feedback も agent workspace にだけ送る。

### 4) 設定と配布
- `reserveTokens` の default は `2048`。
- `postinstall.cjs` は binary が取れないときに hard fail しない。
- `EPISODIC_SKIP_POSTINSTALL=1` で postinstall をスキップできる。
- README 系と manifest の説明は「disabled / ignored」に揃えてある。

## 受け入れ条件
- shared / cross-agent recall の runtime 経路が残っていない。
- `assemble()` の recall cache が誤って共有されない。
- `ep-recall` / `ep-expand` が他 workspace を見ない。
- `npm run build` と `npm test` が通る。
- manifest / README / plan が同じ契約を説明している。

## テスト観点
- 単一 agent での正常 recall と token=0 の早期 return。
- 2 agent 環境でも workspace が混線しないこと。
- `sharedEpisodesDir` / `allowCrossAgentRecall` を設定しても runtime 行動が変わらないこと。
- `ep-save` 後に同じ workspace の cache が更新されること。
- postinstall が offline / binary 不在でも install を落とさないこと。
