# Incident Report: Automatic Episode Generation Stalled Since April 1, 2026

**Date:** 2026-04-05
**Severity:** CRITICAL
**Status:** Root cause confirmed via OpenClaw source audit, v0.3.4 fix plan
**Affected Versions:** v0.2.4 through v0.3.3
**Fix Target:** v0.3.4

---

## 1. Executive Summary

Automatic episode segmentation (Surprise-based topic boundary detection) has been completely non-functional since **2026-04-01 20:38:40 (UTC+7)**. No new episodes have been created through the automatic pipeline for 4+ days. Only manual `ep-save` operations and `ep-recall` reads have been working.

**Root cause confirmed by auditing the OpenClaw source code:** OpenClaw calls `assemble()` on **exactly one** context engine -- the one resolved from `config.plugins.slots.contextEngine`. If this config key is not set to `"episodic-claw"`, OpenClaw falls back to the built-in `"legacy"` engine and episodic-claw's `assemble()` is never invoked.

The production server has not been updated since 2026-03-31. The breakage was caused by an **OpenClaw-side change** (commit `235908c30e` on March 30) that altered multi-kind plugin slot ownership logic, which likely caused episodic-claw to lose the `contextEngine` slot.

## 3. Root Cause Analysis (Confirmed via OpenClaw Source Audit)

### 3.1. The Call Chain Breakpoint

The automatic episode generation pipeline is:

```
OpenClaw run attempt
  -> resolveContextEngine(config)     [src/context-engine/registry.ts L411-426]
    -> reads config.plugins.slots.contextEngine (default: "legacy")
    -> returns the SINGLE resolved context engine
  -> assembleAttemptContextEngine()   [src/agents/pi-embedded-runner/run/attempt.ts L1200-1231]
    -> contextEngine.assemble({ sessionId, messages, tokenBudget, ... })
      -> assemble(ctx)                [episodic-claw src/index.ts L707]
        -> segmenter.processTurn()    [episodic-claw src/index.ts L721, fire-and-forget]
          -> rpc.segmentScore()       [episodic-claw src/segmenter.ts L185]
            -> ai.segmentScore RPC    [Go sidecar]
              -> isBoundary? -> batchIngest() -> episode file written
```

**Breakpoint:** `resolveContextEngine()` does NOT return episodic-claw's engine. It returns `"legacy"` by default.

### 3.2. OpenClaw's Context Engine Resolution (Source-Confirmed)

**File:** `openclaw-source/src/context-engine/registry.ts` lines 411-426

```typescript
export async function resolveContextEngine(config: OpenClawConfig): Promise<ContextEngine> {
  const slotId = config.plugins?.slots?.contextEngine
    ?? defaultSlotIdForKey("contextEngine");
  // defaultSlotIdForKey("contextEngine") returns "legacy"
  const entry = CONTEXT_ENGINE_REGISTRY_STATE.get(slotId);
  if (!entry) throw new Error(`Context engine "${slotId}" not registered`);
  const engine = await entry.factory();
  return wrapContextEngineWithSessionKeyCompat(engine);
}
```

**This means:**
- OpenClaw does NOT call `assemble()` on all registered context engines
- It calls `assemble()` on **exactly one** engine: the one in `config.plugins.slots.contextEngine`
- If not configured, it defaults to `"legacy"` (OpenClaw's built-in engine)
- episodic-claw registers as `"episodic-claw"`, so it is only called if the user explicitly sets `"contextEngine": "episodic-claw"` in their config

### 3.3. The Assemble Call Site (Source-Confirmed)

**File:** `openclaw-source/src/agents/pi-embedded-runner/run/attempt.ts` lines 1200-1231

```typescript
if (params.contextEngine) {
  try {
    const assembled = await assembleAttemptContextEngine({
      contextEngine: params.contextEngine,
      sessionId: params.sessionId,
      sessionKey: params.sessionKey,
      messages: activeSession.messages,
      tokenBudget: params.contextTokenBudget,
      modelId: params.modelId,
      ...(params.prompt !== undefined ? { prompt: params.prompt } : {}),
    });
    // ... uses assembled.messages and assembled.systemPromptAddition
  } catch (assembleErr) {
    log.warn(`context engine assemble failed, using pipeline messages: ${assembleErr}`);
  }
}
```

There is exactly **one guard**: `if (params.contextEngine)`. If truthy, assemble is always called. The `contextEngine` comes from `resolveContextEngine()` in `run.ts` line 391.

### 3.4. Why It Broke on April 1

**The production server has not been updated since 2026-03-31.** The breakage was caused by an OpenClaw-side change:

**Commit `235908c30e` (Mar 30):** "fix: support multi-kind plugins for dual slot ownership"

This commit changed how plugins with multiple kinds (e.g., `["memory", "context-engine"]`) are handled during slot selection and disablement. Before this commit, the disablement logic may have been less strict, allowing episodic-claw to receive assemble calls even without explicit slot configuration. After this commit, the slot resolution became stricter, and episodic-claw lost the `contextEngine` slot to the `"legacy"` default.

**Other relevant commits in the timeframe:**
- `bf0f33db32` (Apr 1): "fix(compaction): resolve model override in runtime context for all context engines" -- only affects `/compact` command, not assemble
- `78e74d4a64` (Mar 31): "Plugins: preserve prompt build system prompt precedence" -- affects hook precedence, not assemble

### 3.5. What Does NOT Cause This (Confirmed)

| Hypothesis | Verdict | Evidence |
|-----------|---------|----------|
| `ownsCompaction: true` removal | **NOT A CAUSE** | `ownsCompaction` is only used for compaction hook behavior, not assemble invocation |
| `before_prompt_build` replacing assemble | **NOT A CAUSE** | `before_prompt_build` runs AFTER assemble (attempt.ts L1469); they are complementary |
| Plugin API compat range | **NOT A CAUSE** | Plugin loads fine; `ep-recall`, `ep-save`, `ep-anchor` all work |
| Dedup early return in processTurn | **NOT A CAUSE** | assemble() is never called, so processTurn() never runs |
| episodic-claw code change | **NOT A CAUSE** | Production server unchanged since Mar 31 |

## 4. HealingWorker Assessment

### 4.1. Current Behavior

The HealingWorker runs on a 30-minute interval and executes:
- **Pass 1**: Check for episodes missing from vector DB, generate embeddings (zero executions since Apr 1)
- **Pass 2**: Refine MD5 slugs to human-readable names via LLM (zero executions since Apr 1)
- **Pass 3**: Batch score update for all episodes (running normally, every 30 min)
- **Pass 4**: Garbage collection of tombstoned episodes older than 14 days (running normally, every 30 min)

### 4.2. Assessment

**The HealingWorker is functioning correctly.** Pass 1 and Pass 2 have no work to do because no new episodes are being created. Pass 3 and Pass 4 run on schedule but operate on a static dataset of 863+ episodes, consuming CPU for ~48 full DB scans per day with no changes to process.

### 4.3. Recommended Optimization

Add a change-detection skip: if no episodes were added/modified/removed since the last pass, skip Pass 3 and Pass 4 for that cycle. This reduces unnecessary CPU usage during idle periods.

## 5. v0.3.4 Fix Plan

### 5.1. Fix A: Watcher-Based Fallback Segmentation (Primary)

Since we cannot rely on `assemble()` being called by OpenClaw, implement a fallback that triggers segmentation from the file watcher or session activity.

**Approach:** Use the existing `before_prompt_build` hook (which IS called by OpenClaw regardless of context engine slot) to trigger `segmenter.processTurn()`.

**File:** `src/index.ts`

```typescript
// Register a before_prompt_build hook as a segmentation trigger fallback
api.on("before_prompt_build", async (event, ctx) => {
  const msgs = (ctx.messages || []) as Message[];
  const resolution = resolveAgentWorkspaces(ctx, openClawGlobalConfig);
  const { agentId, agentWs } = resolution;
  if (!agentWs || !state?.segmenter) return;

  // Fire-and-forget segmentation from prompt build
  state.segmenter.processTurn(msgs, agentWs, agentId).catch(err => {
    console.log("[Episodic Memory] Fallback segmenter error in before_prompt_build:", err);
  });
});
```

**Why this works:** The `before_prompt_build` hook is called by OpenClaw on every prompt build (attempt.ts L1469-1523), regardless of which context engine is selected. This gives us a reliable trigger point for segmentation that does not depend on the `contextEngine` slot configuration.

### 5.2. Fix B: Diagnostic Logging

Add logging to confirm whether `assemble()` is being called and why segmentation is or isn't happening:

```typescript
// In assemble():

// In processTurn() early returns (src/segmenter.ts):
```

### 5.3. Fix C: Config Validation Warning

Add a startup check that warns if the `contextEngine` slot is not set to `"episodic-claw"`:

```typescript

console.log("[Episodic Memory] If assemble() is not being called, use the before_prompt_build fallback (enabled by default in v0.3.4+)");
```

### 5.4. Fix D: HealingWorker Optimization

Add a "no changes" skip to the HealingWorker's Pass 3/4 cycle:

```go

    log.Debug("Skipping Pass 3/4: no workspace changes detected")
    return
}
```
