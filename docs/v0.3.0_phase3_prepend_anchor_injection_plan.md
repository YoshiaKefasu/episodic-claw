# v0.3.0 Phase 3 プラン - prependSystemContext への一時 Anchor 注入

- 状態: 収束済み

## 目的
- compaction 直後だけ `Anchor + compaction summary` を `prependSystemContext` へ注入する。
- recall 注入と Anchor 注入を混ぜずに、source を分けて扱う。

## このフェーズで決めること
- compaction 直後の一時注入 state をどう寿命管理するか
- `anchorInjectionAssembles` をどの単位で消費するか
- recall 注入と Anchor 注入をどう分離して追跡するか
- ログの必須項目を何にするか

## 現行コードとの関係
- [index.ts](/D:/GitHub/OpenClaw%20Related%20Repos/episodic-claw/src/index.ts) は `assemble()` から `prependSystemContext` を返している。
- 現行コードにはすでに `AnchorInjectionState`、`AgentRuntimeState.anchorInjection`、`activateAnchorInjection()`、`assemble()` 内の注入・寿命減算・ログ、`compact()` 後の state 活性化がある。
- `prependSystemContext` は recall outcome と Anchor injection を 1 本の文字列に束ねて返す。
- [retriever.ts](/D:/GitHub/OpenClaw%20Related%20Repos/episodic-claw/src/retriever.ts) の recall 注入と混ざると、原因切り分けが難しくなる。

## 実装確認ポイント
- `before_prompt_build` 主体の merge path と、recall / Anchor の separation が実コードと一致しているか。
- `compact()` が残した `result.anchor` / `result.summary` を runtime state へ受け渡す境界が実装済みか。
- ログに `status`, `source`, `anchorId`, `summaryId`, `estimatedTokens`, `anchorInjectionWindow` が残っているか。

## 設計方針
- 常時注入はしない。
- compaction の橋渡し期間だけ注入する。
- Phase 3 の寿命設定は `anchorInjectionAssembles`（デフォルト 1）で切る。これは **生の assemble 回数** ではなく、Anchor 注入判定に進んだ eligible assemble で消費される。
- recall は recall、Anchor は Anchor で別 source とする。
- token budget で切り詰めたときも、Anchor の注入結果は別 reason で追えるようにする。

## 受け入れ条件
- compaction 直後だけ Anchor と summary が注入される。
- 寿命切れ後は自動で注入が止まる。
- recall 注入と Anchor 注入のログを区別できる。
- 注入の有無と理由が運用ログで追える。
- `after_compaction` は通知用 hook であり、Anchor payload の受け渡しはしない。Anchor state の活性化は plugin の `compact()` 成功後に行う。

## 実装状況
- `AnchorInjectionState` と `anchorInjection` state は実装済み。
- `compact()` の後に `activateAnchorInjection()` が呼ばれ、次の `assemble()` で `prependSystemContext` に合成される。
- 上記 `compact()` 起点は host native `/compact` 経路（`compactEmbeddedPiSession -> contextEngine.compact()`）でも同じ。
- `budget_truncated_to_zero` の早期 return では寿命を消費しない。
- `before_prompt_build` 主体、`before_agent_start` fallback の merge path は OpenClaw host 契約と整合している。

## 非対象
- Anchor の中身生成
- raw history の保存
- `ep-save` や手動保存の入口
- lexical 更新そのもの

## Research Addendum — 2026-04-03

### Findings
- OpenClaw host での最終 merge 点は `composeSystemPromptWithHookContext()` です。`prependSystemContext` は `baseSystemPrompt` の前に、`appendSystemContext` は後ろに結合されます。根拠: `D:\GitHub\OpenClaw Related Repos\openclaw-source\src\agents\pi-embedded-runner\run\attempt.thread-helpers.ts:6-20`
- hook 実行順は `resolvePromptBuildHookResult()` → `before_prompt_build` → legacy fallback の `before_agent_start` で、`prependSystemContext` は両方の結果を `joinPresentTextSegments()` で合流させます。根拠: `D:\GitHub\OpenClaw Related Repos\openclaw-source\src\agents\pi-embedded-runner\run\attempt.prompt-helpers.ts:28-81`
- host 側では、`systemPrompt` override が先に適用され、そのあと `composeSystemPromptWithHookContext()` で `prependSystemContext` / `appendSystemContext` が最終合成されます。根拠: `D:\GitHub\OpenClaw Related Repos\openclaw-source\src\agents\pi-embedded-runner\run\attempt.ts:1496-1520`
- prompt injection の public hook 名は `before_prompt_build` と `before_agent_start` だけです。`prependSystemContext` / `prependContext` は `PluginHookBeforePromptBuildResult` のフィールドとして定義されています。根拠: `D:\GitHub\OpenClaw Related Repos\openclaw-source\src\plugins\types.ts:1999-2002`, `D:\GitHub\OpenClaw Related Repos\openclaw-source\src\plugins\types.ts:2070-2079`, `D:\GitHub\OpenClaw Related Repos\openclaw-source\src\plugins\types.ts:2117-2137`
- `episodic-claw` の現状 `assemble()` は recall 注入と Anchor 注入を同じ `prependSystemContext` へ合成し、ログも recall 向けと anchor 向けを別 `status/reason` で出し分けています。根拠: `D:\GitHub\OpenClaw Related Repos\episodic-claw\src\index.ts:102-126`, `D:\GitHub\OpenClaw Related Repos\episodic-claw\src\index.ts:530-779`
- `episodic-claw` の runtime state には、`recallCache` に加えて `anchorInjection` があり、`compact()` 後に `activateAnchorInjection()` で有効化されます。根拠: `D:\GitHub\OpenClaw Related Repos\episodic-claw\src\index.ts:85-106`, `D:\GitHub\OpenClaw Related Repos\episodic-claw\src\index.ts:456-475`, `D:\GitHub\OpenClaw Related Repos\episodic-claw\src\index.ts:756-779`

- host の `before_compaction` / `after_compaction` は void hook で、engine-owned compaction でも通知されますが、`after_compaction` event には `anchor` / `summary` は載りません。`messageCount` と `compactedCount` は engine-owned path では `-1` が渡されます。根拠: `D:\GitHub\OpenClaw Related Repos\openclaw-source\src\plugins\types.ts:2231-2259`, `D:\GitHub\OpenClaw Related Repos\openclaw-source\src\plugins\hooks.ts:567-583`, `D:\GitHub\OpenClaw Related Repos\openclaw-source\src\agents\pi-embedded-runner\compact.ts:1036-1144`

### Implementation-prep Notes
- Phase 3 の one-shot / time-limited 注入 state は、`AgentRuntimeState` の隣に置くのが最小です。理由は、いまの `assemble()` と `compact()` がどちらも同じ agent-local state を通っているからです。根拠: `D:\GitHub\OpenClaw Related Repos\episodic-claw\src\index.ts:91-106`, `D:\GitHub\OpenClaw Related Repos\episodic-claw\src\index.ts:530-779`
- `prependSystemContext` は host 側で **単一の string** として consume されるため、Anchor と summary を個別に持っていても、実際に返す直前で 1 本の注入文字列へ束ねる必要があります。根拠: `D:\GitHub\OpenClaw Related Repos\openclaw-source\src\agents\pi-embedded-runner\run\attempt.thread-helpers.ts:6-20`
- 最小の `AnchorInjectionState` は、少なくとも `anchorText`, `summaryText`, `anchorId`, `summaryId`, `remainingEligibleAssembles`, `source` を持てば足ります。`estimatedTokens` は state に保持せず、ログ時に `estimateTokens(...)` で導出する今の形が最小です。根拠: `D:\GitHub\OpenClaw Related Repos\episodic-claw\src\index.ts:91-98`, `D:\GitHub\OpenClaw Related Repos\episodic-claw\src\index.ts:141-163`, `D:\GitHub\OpenClaw Related Repos\episodic-claw\src\index.ts:680-702`
- recall 注入と混ぜないために、Phase 3 では `RecallInjectionOutcome` を拡張するより、別 source の注入判定と別ログ行に分ける方が現行構造に合っています。根拠: `D:\GitHub\OpenClaw Related Repos\episodic-claw\src\retriever.ts:9-16`, `D:\GitHub\OpenClaw Related Repos\episodic-claw\src\index.ts:597-779`
- `before_agent_start` は legacy fallback として `prependSystemContext` に合流しますが、Phase 3 の一時 Anchor 注入は per-turn assemble 判定なので、主経路は `before_prompt_build` 相当の `assemble()` 側に置くのが source 分離に向いています。根拠: `D:\GitHub\OpenClaw Related Repos\openclaw-source\src\agents\pi-embedded-runner\run\attempt.prompt-helpers.ts:35-81`, `D:\GitHub\OpenClaw Related Repos\episodic-claw\src\index.ts:530-614`

### Lines Needing Correction Before Implementation
- `anchorInjectionAssembles` は生の assemble 回数ではなく、Anchor 注入判定に到達した eligible assemble で消費される現在仕様として書くと、`budget_truncated_to_zero` の早期 return とも矛盾しません。
- `after_compaction` は通知用 hook であり、payload を渡す場所ではない、と明示しておくと host 側責務とのズレを防げます。

### Resolution
- Phase 3 の merge point、hook 契約、現状 state、compaction 結果の受け渡し境界はそろっています。
- 寿命は `anchorInjectionAssembles` の eligible-assemble semantics で統一され、Anchor と recall の責務分離も実装と整合しています。
- `after_compaction` を通知用 hook として扱う書き方に寄せたことで、Phase 3 文書は現行コードと矛盾しない状態に収束しました。
